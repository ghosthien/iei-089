🪟 ¿Qué es Windows Forms?

Antes de escribir código, entendamos qué es lo que vamos a usar y por qué existe.

📐 Definición técnica
Windows Forms (WinForms) es un framework de la BCL de .NET para crear interfaces gráficas de escritorio en Windows. Cada ventana es un objeto de tipo Form. Los elementos visuales (botones, campos, tablas) son objetos de tipo Control que se agregan al formulario con Controls.Add().
💡 En palabras simples
Hasta ahora el programa solo mostraba texto en una pantalla negra. Con Windows Forms, el programa abre una ventana de Windows real con botones, campos para escribir y tablas — exactamente como cualquier software que usas en tu PC. El código C# controla todo lo que pasa en esa ventana.
🏢 Ejemplo profesional
Falabella, Ripley y Paris usan sistemas internos de punto de venta construidos con Windows Forms. El cajero ve una ventana con campos para ingresar el código del producto y botones para procesar la venta — exactamente lo que construiremos en Almacenes Por Ahí.
🔄Consola vs Windows Forms — comparación
📟 App de Consola (Semanas 1-2)
  • Solo texto — sin botones ni ventanas
  • El usuario escribe con el teclado y presiona Enter
  • La interfaz se construye con Console.WriteLine()
  • Un solo hilo de ejecución de arriba hacia abajo
  • Útil para aprender lógica y colecciones
🪟 Windows Forms (Semana 3+)
  • Ventana real con botones, campos y tablas
  • El usuario hace clic — el programa reacciona a eventos
  • La interfaz se construye con controles (new Button())
  • Programación orientada a eventos: el código espera acciones
  • Reutiliza los servicios que ya escribiste en la consola

XML — AlmacenesWinForms.csproj (archivo de configuración) Paso 3 
<!-- Este archivo configura el proyecto Windows Forms -->
<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <!-- Exe = genera un .exe ejecutable (no librería) -->
    <OutputType>WinExe</OutputType>

    <!-- .NET 8 con soporte para Windows (necesario para WinForms) -->
    <TargetFramework>net8.0-windows</TargetFramework>

    <!-- ESTA ES LA LÍNEA CLAVE: activa Windows Forms -->
    <UseWindowsForms>true</UseWindowsForms>

    <!-- Habilita anotaciones de null (detecta errores de null) -->
    <Nullable>enable</Nullable>

    <!-- Agrega using System; y otros namespaces automáticamente -->
    <ImplicitUsings>enable</ImplicitUsings>
  </PropertyGroup>

</Project>

C# — Program.cs de Windows Forms Paso 4 
// Program.cs de Windows Forms — generado automáticamente
// Es diferente al Program.cs de la consola

// ApplicationConfiguration.Initialize():
// Configura DPI (resolución de pantalla), fuentes y compatibilidad
// SIEMPRE debe ser la primera línea — sin esto, la ventana puede verse borrosa
ApplicationConfiguration.Initialize();

// Application.Run(form):
// Abre el formulario especificado y mantiene la app en ejecución
// La app corre mientras esta ventana esté abierta
// Cuando el usuario cierra FrmPrincipal → la app termina
Application.Run(new FrmPrincipal());

// Diferencia con la consola:
// Consola:  el código corre de arriba hacia abajo y termina
// WinForms: Application.Run() entra en un BUCLE de eventos interno
//           que espera clics, teclazos, etc. hasta que el usuario cierre

🧩 Controles Esenciales de Windows Forms

Los controles son los elementos visuales de la interfaz — botones, campos, tablas, listas. Cada uno es un objeto C# con propiedades y eventos.

¿Qué es un Control?
Un control es un objeto C# que tiene representación visual en la pantalla. Tiene propiedades que definen su apariencia (Text, Size, Location), y eventos a los que puede reaccionar (Click, TextChanged). Para que aparezca en la ventana debes agregarlo con Controls.Add(miControl) — sin esa línea el control existe en memoria pero no se ve.
🏷️
Label
Texto estático no editable. Funciona como una etiqueta que le dice al usuario qué datos ingresar.
📍 Todos los formularios
📝
TextBox
Campo de texto editable. El usuario escribe aquí. Se lee con .Text y se limpia con .Clear().
📍 FrmInventario, FrmClientes
🔘
Button
Botón que reacciona al clic. Conectado a un método manejador de eventos con el operador +=.
📍 Todos los formularios
📊
DataGridView
Tabla con filas y columnas. Muestra colecciones de datos. Se rellena con Rows.Add() y se limpia con Rows.Clear().
📍 FrmInventario, FrmClientes
📋
ComboBox
Lista desplegable de opciones. El usuario elige una opción. Se llena con Items.Add().
📍 FrmVentas (producto y cliente)
🔢
NumericUpDown
Campo numérico con flechas ↑↓ para subir/bajar el valor. Evita que el usuario ingrese texto no numérico.
📍 FrmVentas (cantidad)
📦
GroupBox
Contenedor que agrupa controles relacionados con un borde y título visual. Organiza el formulario.
📍 FrmInventario, FrmClientes
💬
MessageBox
Diálogo emergente de validación. Método estático — no se agrega al form. Bloquea hasta que el usuario hace clic.

⚡ Eventos — Cómo Reacciona la Ventana

Un evento es algo que ocurre: el usuario hace clic, presiona una tecla, cierra la ventana. El código C# "escucha" esos eventos y reacciona ejecutando un método.

📐 Definición técnica
Un evento es un mecanismo del lenguaje C# basado en el patrón Observer (delegados). btn.Click += Metodo usa el operador += para suscribir un método al evento Click. Cuando ocurre el clic, el runtime invoca automáticamente todos los métodos suscritos.
💡 En palabras simples
Es como poner un timbre en una puerta. El timbre es el evento (Click). El operador += conecta el timbre con una acción (el método). Cuando alguien presiona el timbre (clic del usuario), la acción se ejecuta automáticamente — sin que tú llames al método manualmente.
C# — Eventos: Click, TextChanged y SelectedIndexChanged
// ─── EVENTO CLICK ─────────────────────────────────────────
// Sintaxis: control.Click += NombreDelMetodo;
// El método manejador siempre tiene estos dos parámetros:
//   object? sender → el control que disparó el evento (el botón)
//   EventArgs e    → datos del evento (para Click no se usa mucho)

// 1. Suscribir el método al evento (esto va en el constructor del form)
btnAgregar.Click += BtnAgregar_Click; // += = "cuando ocurra Click, llama a BtnAgregar_Click"
btnBuscar.Click  += BtnBuscar_Click;
btnLimpiar.Click += BtnLimpiar_Click;

// 2. Definir el método manejador (private, void, con los dos parámetros)
private void BtnAgregar_Click(object? sender, EventArgs e)
{
    // Validar que los campos no estén vacíos ANTES de procesar
    // string.IsNullOrWhiteSpace() → true si es null, vacío o solo espacios
    if (string.IsNullOrWhiteSpace(_txtNombre.Text))
    {
        MessageBox.Show("El nombre es obligatorio.", "Validación",
            MessageBoxButtons.OK, MessageBoxIcon.Warning);
        _txtNombre.Focus(); // poner el cursor en el campo vacío
        return; // salir del método sin procesar
    }

    // Convertir el TextBox (string) a decimal con TryParse
    if (!decimal.TryParse(_txtPrecio.Text, out decimal precio) || precio <= 0)
    {
        MessageBox.Show("Ingresa un precio válido mayor a 0.", "Validación",
            MessageBoxButtons.OK, MessageBoxIcon.Warning);
        return;
    }

    // Llamar al servicio (mismo que usábamos en la consola)
    _svc.Agregar(_txtNombre.Text, precio, (int)_nudStock.Value);

    // Refrescar la tabla con los datos actualizados
    CargarGrid();

    // Limpiar los campos para el próximo ingreso
    _txtNombre.Clear();
    _txtPrecio.Clear();
    _nudStock.Value = 0;
    _txtNombre.Focus(); // cursor vuelve al primer campo
}

// ─── EVENTO SELECTEDINDEXCHANGED (ComboBox) ───────────────
// Se dispara cuando el usuario cambia la selección
private void CmbProducto_SelectedIndexChanged(object? sender, EventArgs e)
{
    if (_cmb.SelectedItem is string seleccionado)
        Console.WriteLine($"Seleccionado: {seleccionado}"); // para debug
}

// ─── EVENTOS MÁS COMUNES ─────────────────────────────────
// Click             → cuando se hace clic en un control
// TextChanged       → cuando cambia el texto de un TextBox
// SelectedIndexChanged → cuando cambia la selección de un ComboBox
// FormClosing       → justo antes de que se cierre el formulario
// Load              → cuando el formulario termina de cargarse

📋 Formulario Completo — FrmInventario

Estructura completa del formulario de inventario. Este es el patrón que se repite en todos los formularios del proyecto.

¿Por qué el formulario hereda de Form?
Cuando escribes public class FrmInventario : Form, estás diciendo que FrmInventario hereda de la clase Form. Eso significa que tu formulario recibe automáticamente todas las propiedades (como Text para el título, Size para el tamaño) y los métodos de Form. Sin esa herencia, no sería una ventana. Es exactamente igual a como en Java extiendes JFrame.
C# WinForms — FrmInventario.cs completo y comentado
// Archivo: Forms/FrmInventario.cs
// Formulario de inventario — muestra, agrega y busca productos
using System.Windows.Forms; // namespace de todos los controles WinForms
namespace AlmacenesWinForms.Forms;

// FrmInventario hereda de Form → obtiene todas las propiedades de una ventana
public class FrmInventario : Form
{
    // ─── DEPENDENCIA INYECTADA ────────────────────────────────
    // El servicio viene de la consola — no se duplica código
    // readonly: la referencia no cambia una vez asignada en el constructor
    private readonly ProductoService _svc;

    // ─── CONTROLES (campos privados) ──────────────────────────
    // Se declaran aquí para que todos los métodos de la clase puedan usarlos
    // null! → le decimos al compilador "sé que es null ahora, pero en BuildUI() lo asignaré"
    private DataGridView _grid    = null!;
    private TextBox      _txtNombre = null!;
    private TextBox      _txtPrecio = null!;
    private NumericUpDown _nudStock = null!;

    // ─── CONSTRUCTOR ─────────────────────────────────────────
    public FrmInventario(ProductoService svc)
    {
        _svc = svc;

        // Propiedades del formulario (la ventana)
        Text          = "Inventario de Productos — Almacenes Por Ahí"; // título
        Size          = new Size(800, 560);                         // tamaño ventana
        StartPosition = FormStartPosition.CenterScreen;             // centrar en pantalla
        MinimumSize   = new Size(600, 400);                         // tamaño mínimo

        BuildUI();      // crear y posicionar todos los controles
        CargarGrid();   // llenar la tabla con los datos actuales
    }

    // ─── BUILD UI ─────────────────────────────────────────────
    // Este método crea todos los controles y los agrega al formulario
    private void BuildUI()
    {
        // Panel superior — formulario de ingreso
        GroupBox grp = new GroupBox
        {
            Text     = "Agregar Producto",
            Location = new Point(10, 10),
            Size     = new Size(765, 90)
        };
        Controls.Add(grp);

        // Nombre
        grp.Controls.Add(new Label { Text = "Nombre:", Location = new Point(10, 25), AutoSize = true });
        _txtNombre = new TextBox { Location = new Point(65, 22), Size = new Size(250, 24) };
        grp.Controls.Add(_txtNombre);

        // Precio
        grp.Controls.Add(new Label { Text = "Precio:", Location = new Point(330, 25), AutoSize = true });
        _txtPrecio = new TextBox { Location = new Point(380, 22), Size = new Size(100, 24), PlaceholderText = "9990" };
        grp.Controls.Add(_txtPrecio);

        // Stock con NumericUpDown (flechas para subir/bajar)
        grp.Controls.Add(new Label { Text = "Stock:", Location = new Point(495, 25), AutoSize = true });
        _nudStock = new NumericUpDown { Location = new Point(540, 22), Size = new Size(70, 24), Minimum = 0, Maximum = 9999 };
        grp.Controls.Add(_nudStock);

        // Botón Agregar
        Button btnAgregar = new Button { Text = "➕ Agregar", Location = new Point(630, 20), Size = new Size(120, 32) };
        btnAgregar.Click += BtnAgregar_Click; // conectar evento
        grp.Controls.Add(btnAgregar);

        // DataGridView — tabla de productos
        _grid = new DataGridView
        {
            Location             = new Point(10, 115),
            Size                 = new Size(765, 340),
            ReadOnly             = true,
            AllowUserToAddRows   = false,
            SelectionMode        = DataGridViewSelectionMode.FullRowSelect,
            AutoSizeColumnsMode  = DataGridViewAutoSizeColumnsMode.Fill
        };
        _grid.Columns.Add("colId",     "ID");
        _grid.Columns.Add("colNombre", "Nombre");
        _grid.Columns.Add("colPrecio", "Precio");
        _grid.Columns.Add("colStock",  "Stock");
        Controls.Add(_grid); // ← SIN ESTO LA TABLA NO APARECE
    }

    // ─── EVENTO HANDLERS ─────────────────────────────────────
    private void BtnAgregar_Click(object? sender, EventArgs e)
    {
        // Validar nombre
        if (string.IsNullOrWhiteSpace(_txtNombre.Text))
        {
            MessageBox.Show("El nombre es obligatorio.", "Validación",
                MessageBoxButtons.OK, MessageBoxIcon.Warning);
            return;
        }

        // Validar precio con TryParse (NUNCA usar decimal.Parse en un TextBox)
        if (!decimal.TryParse(_txtPrecio.Text, out decimal precio) || precio <= 0)
        {
            MessageBox.Show("Ingresa un precio válido mayor a 0.", "Validación",
                MessageBoxButtons.OK, MessageBoxIcon.Warning);
            return;
        }

        // Todo válido → llamar al servicio (misma lógica que en la consola)
        _svc.Agregar(_txtNombre.Text.Trim(), precio, (int)_nudStock.Value);

        CargarGrid(); // actualizar la tabla
        _txtNombre.Clear(); _txtPrecio.Clear(); _nudStock.Value = 0;
        _txtNombre.Focus();
    }

    // ─── LLENAR DATAGRIDVIEW ──────────────────────────────────
    private void CargarGrid()
    {
        _grid.Rows.Clear(); // limpiar filas existentes
        foreach (var p in _svc.Listar()) // mismo servicio de la consola
            _grid.Rows.Add(p.Id, p.Nombre, $"${p.Precio:N0}", p.Stock);
    }
}

🔀 Navegación entre Formularios

Una aplicación real tiene varias ventanas. Aprende a abrirlas correctamente.

ShowDialog() — bloqueante
Abre la ventana hija y bloquea el formulario padre. El usuario no puede interactuar con la ventana padre hasta que cierre la hija. Usar para submenús y ventanas de detalle donde el padre debe esperar.
Show() — no bloqueante
Abre la ventana hija pero el padre sigue activo. Ambas ventanas son independientes. Usar para paneles que pueden estar abiertos en paralelo, como un panel de monitoreo.
C# WinForms — Navegación desde FrmPrincipal
// FrmPrincipal.cs — menú principal de la aplicación
public class FrmPrincipal : Form
{
    // El servicio se crea UNA VEZ aquí y se pasa a cada formulario hijo
    // Así todos comparten los mismos datos en memoria
    private readonly ProductoService _productoSvc = new();

    public FrmPrincipal()
    {
        Text = "Almacenes Por Ahí — Sistema de Gestión";
        Size = new Size(500, 350);
        StartPosition = FormStartPosition.CenterScreen;
        BuildUI();
    }

    private void BuildUI()
    {
        Button btnInventario = new Button { Text = "📦 Inventario", Location = new Point(150, 80), Size = new Size(200, 50) };
        btnInventario.Click += BtnInventario_Click;
        Controls.Add(btnInventario);

        Button btnClientes  = new Button { Text = "👥 Clientes",   Location = new Point(150, 150), Size = new Size(200, 50) };
        btnClientes.Click  += BtnClientes_Click;
        Controls.Add(btnClientes);
    }

    private void BtnInventario_Click(object? sender, EventArgs e)
    {
        // ShowDialog: bloquea FrmPrincipal mientras FrmInventario está abierto
        // this = pasar referencia al padre (buena práctica)
        // El servicio se pasa como parámetro — mismo objeto en memoria
        using var frm = new FrmInventario(_productoSvc);
        frm.ShowDialog(this);
        // El código continúa acá DESPUÉS de que FrmInventario se cierre
    }

    private void BtnClientes_Click(object? sender, EventArgs e)
    {
        using var frm = new FrmClientes();
        frm.ShowDialog(this);
    }
}
⚠️ Diferencia clave con Java
En Java (Swing/JavaFX) se usa setVisible(true) y el padre sigue activo. En WinForms, para bloquear el padre hay que usar ShowDialog(). Usar solo Show() en WinForms hace que ambas ventanas estén activas al mismo tiempo — eso puede confundir al usuario en un flujo de negocio lineal.

6
Errores frecuentes y cómo resolverlos
Troubleshooting
❌ El control no aparece en la ventana
Falta Controls.Add(miControl). Puedes crear el control, configurarlo, pero si no lo agregas al formulario con Controls.Add() no será visible.
❌ NullReferenceException al cargar
El orden en el constructor importa: primero BuildUI() (que asigna los controles) y después CargarGrid() (que los usa). Si está al revés, los controles son null.
❌ Error de compilación en .csproj
Verifica que <UseWindowsForms>true</UseWindowsForms> esté en el .csproj y que el TargetFramework sea net8.0-windows (con -windows).
❌ decimal.Parse lanza excepción
Nunca usar decimal.Parse(textBox.Text) — si el usuario escribe texto el programa se cae. Siempre usar decimal.TryParse(textBox.Text, out decimal valor).

📖 Glosario — Windows Forms

Busca cualquier término técnico de la clase. +2 XP por cada término explorado.

Form — La VentanaWinForms

Técnica: Clase de System.Windows.Forms que representa una ventana del sistema operativo. Hereda propiedades como Text (título), Size, BackColor.

Simple: Cada ventana que ve el usuario es un objeto de tipo Form. Tu clase la hereda: class FrmMiVentana : Form.

Java: Equivale a extender JFrame en Swing.

Control — Elemento VisualWinForms

Técnica: Clase base de todos los elementos visuales (Button, TextBox, Label, etc.). Tiene propiedades Location, Size, Visible y el método Focus().

Simple: Cualquier cosa que ves en la ventana es un Control. Para que aparezca debes llamar Controls.Add(miControl) — sin eso no existe visualmente.

Evento — Click y manejadoresWinForms

Técnica: Mecanismo basado en delegados. btn.Click += Metodo suscribe el método al evento. Cuando ocurre el Click, el runtime invoca todos los métodos suscritos.

Simple: Es como poner un timbre: += conecta el timbre (evento) con la acción (método). Cuando el usuario hace clic → tu método se ejecuta automáticamente.

Java: Equivale a btn.addActionListener(e -> miMetodo());

ShowDialog() vs Show()WinForms

ShowDialog(): Abre la ventana hija en modo modal — bloquea al padre. El código del padre espera hasta que se cierre la hija. Usar para flujos de negocio lineales.

Show(): Abre la ventana hija sin bloquear — el padre sigue activo. Ambas ventanas son independientes.

DataGridView — Tabla VisualWinForms

Técnica: Control de WinForms que muestra datos en filas y columnas. Se configura con Columns.Add() y se llena con Rows.Add(). Rows.Clear() borra todas las filas.

Simple: Es como una tabla de Excel dentro de tu app. Cada Rows.Add(v1,v2,v3) agrega una fila con 3 celdas.

MessageBox.Show() — Diálogo de alertaWinForms

Técnica: Método estático de la clase MessageBox. No se agrega a Controls — se llama directamente. Bloquea la ejecución hasta que el usuario hace clic en el botón del diálogo.

Simple: El cuadrito que aparece con un mensaje y un botón OK. Se usa para validaciones y errores. No necesitas crear un objeto — se llama directo con MessageBox.Show("mensaje").

null! — Supresión de nullC#

Técnica: El operador ! (null-forgiving) suprime la advertencia del compilador cuando una variable nullable se usa en un contexto no-nullable. Se usa cuando el desarrollador sabe que la variable se asignará antes de usarse.

Simple: private TextBox _txt = null!; le dice al compilador: "Sé que ahora es null, pero te prometo que lo asignaré en BuildUI() antes de usarlo". Sin el !, el compilador daría una advertencia.
