📚 FCL Framework Class Library

💬 Analogía: El menú básico que siempre está disponible, sin importar la sucursal donde estés.

⚙️ CLR Common Language Runtime

¿Qué es? El motor de ejecución de .NET. Gestiona: compilación JIT, Garbage Collection, seguridad de tipos y manejo de excepciones.

💬 Analogía: El gerente de operaciones que se asegura que todo se ejecute correctamente.

🔗 Java equivalente: JVM (Java Virtual Machine) — mismo rol, diferente ecosistema.

🖥️ Sistema Operativo Windows / Linux / macOS

¿Qué hace? Gestiona el hardware, la memoria física y los procesos del computador.

🏬 Analogía Almacenes Por Ahí: Los cimientos de hormigón. Sin esto, nada existe. El CLR se apoya sobre el SO para acceder al CPU y la memoria real.

🔗El CLS — Por qué .NET habla varios lenguajes
🔗 CLS — Common Language Specification
Conjunto de reglas que todos los lenguajes .NET deben respetar para garantizar la interoperabilidad. Permite que código en C#, VB.NET y F# conviva y se llame mutuamente dentro del mismo proyecto.
🏢 Ejemplo profesional
El equipo antiguo programó contabilidad en VB.NET. El equipo nuevo (tú) programa en C#. Gracias al CLS, tu módulo C# llama directamente a las clases VB.NET sin reescribir nada. Ventaja real en empresas con sistemas legados.

⚙️ Los 7 Pilares de .NET

Definición técnica, simple y ejemplo profesional por cada pilar. Haz clic para expandir. +5 XP por pilar.

⚙️CLR — Common Language Runtime Motor de ejecución administrado▼
📐 Definición técnica
Entorno de ejecución administrado de .NET. Gestiona el código CIL: lo carga, lo compila JIT, administra la memoria (GC), controla excepciones y aplica seguridad.
💡 Definición simple
El CLR es el "gerente de operaciones" de tu programa. Tú escribes las instrucciones, el CLR se asegura de que se ejecuten correctamente.

Ya conoces este concepto: el CLR es exactamente la JVM de Java. Mismo rol, diferente nombre.
🏬 Ejemplo profesional — Almacenes Por Ahí
Con 50 empleados simultáneos usando el sistema, el CLR garantiza que cada proceso tenga su memoria aislada y que los errores de uno no afecten a los demás.
📦BCL — Base Class Library Clases base reutilizables▼
📐 Definición técnica
Conjunto de clases fundamentales disponibles en cualquier implementación de .NET. Provee tipos base (System), colecciones, texto, matemáticas, I/O y threading.
💡 Definición simple
El "cajón de herramientas básico" siempre disponible. En Java sería java.lang, java.util y java.io combinados.
C# — BCL en Almacenes Por Ahí
using System;                     // Console, DateTime, Math, String
using System.Collections.Generic; // List<T>, Dictionary<K,V>, Queue<T>
using System.Linq;               // Consultas sobre colecciones

// DateTime: tipo de BCL para fechas de venta
DateTime fechaVenta = DateTime.Now;
Console.WriteLine($"Venta: {fechaVenta:dd/MM/yyyy HH:mm}");
🔨Compiladores — Roslyn C# y VB.NET Compiler Platform▼
📐 Definición técnica
Roslyn es el compilador open-source de C# y VB.NET. Transforma el código fuente (.cs) en CIL (.dll / .exe). Provee APIs que permiten analizar y generar código en tiempo real (IntelliSense).
💡 Definición simple
El "traductor" que convierte tu C# legible en el lenguaje que el CLR entiende (CIL). Es el javac de Java, pero además alimenta el autocompletado de Visual Studio.
💡 Dato clave
Al presionar Ctrl+Shift+B en Visual Studio, Roslyn compila. El subrayado rojo en tiempo real también es Roslyn analizando tu código mientras escribes.
📚FCL — Framework Class Library Biblioteca completa de APIs▼
📐 Definición técnica
BCL + todas las APIs específicas de plataforma. Incluye System.Data (ADO.NET), System.Web (ASP.NET), System.Windows (WPF), System.Net, System.IO, System.Security, etc.
💡 Definición simple
El supermercado de herramientas completo. La BCL es el pasillo básico (siempre disponible), la FCL incluye todos los pasillos especializados según el tipo de app.
🔧Herramientas — Visual Studio IDE, CLI, NuGet, dotnet▼
📐 Definición técnica
Ecosistema de herramientas: Visual Studio (IDE completo), VS Code (editor ligero), dotnet CLI (línea de comandos), NuGet (gestor de paquetes) y MSBuild (sistema de compilación).
💡 Definición simple
Visual Studio = el taller completo. NuGet = la ferretería de librerías. dotnet CLI = las instrucciones escritas paso a paso.
🔧 Comandos dotnet CLI
dotnet new console — Crear proyecto · dotnet build — Compilar · dotnet run — Ejecutar · dotnet add package X — Agregar librería NuGet
🌐ASP.NET / ASP.NET Core Framework para aplicaciones web▼
📐 Definición técnica
Framework web multiplataforma open-source. ASP.NET Core soporta MVC, Razor Pages, Web API y Blazor. Se estudiará en la Unidad II (Semanas 7–12).
💡 Definición simple
El "piso web" de .NET. Así como WPF hace apps de escritorio, ASP.NET hace apps que corren en el navegador. Lo usarás en el Encargo 3.
🔁Servicios de Windows Background workers y servicios▼
📐 Definición técnica
Aplicaciones .NET que corren en segundo plano como servicios del sistema operativo. Útiles para tareas programadas, procesamiento asíncrono y monitoreo continuo sin interfaz.
💡 En Almacenes Por Ahí
Un servicio de Windows podría enviar alertas de stock bajo cada hora, o sincronizar inventario entre sucursales durante la noche sin intervención manual.

⚡ JIT — Just-In-Time Compiler

Compila cada método de CIL a código nativo la primera vez que se invoca. El resultado se cachea — llamadas posteriores usan el código ya compilado (más rápido).

C# — El JIT compila este método al primer uso
// Primera llamada → JIT compila → ejecuta → cachea código nativo
// Llamadas siguientes → usa código nativo cacheado (ultrarrápido)
static decimal CalcularDescuento(decimal precio, int pct)
{
    return precio * (1 - pct / 100m);
}
decimal p1 = CalcularDescuento(24990m, 20); // JIT compila aquí
decimal p2 = CalcularDescuento(9990m,  10);  // usa caché nativo

🗑️ Garbage Collector Generacional

Opera en 3 generaciones. Los objetos nuevos (corta vida) van a Gen 0 y se recolectan frecuentemente. Los que sobreviven ascienden a Gen 1 y luego Gen 2.

Gen 0 — Nuevos
Recolección frecuente. Ej: objeto Venta creado para procesar una transacción.
Gen 1 — Intermedios
Sobrevivieron Gen 0. Ej: lista de productos en pantalla.
Gen 2 — Larga vida
Persistentes. Ej: contexto de base de datos de la sesión.

📥 Class Loader

Carga los ensamblados (.dll) desde disco a memoria, verifica su integridad, resuelve dependencias y los hace disponibles para el JIT.

🏬 En práctica: Cuando tu programa arranca, el Class Loader carga AlmacenesConsola.dll y todas sus dependencias automáticamente.

⚠️ Exception Handler

C# — Manejo de excepciones estructurado
try
{
    // Código que podría fallar
    Console.Write("Ingrese precio: ");
    decimal precio = decimal.Parse(Console.ReadLine()); // puede fallar
    Console.WriteLine($"Precio: ${precio:N0}");
}
catch (FormatException)
{
    // El usuario escribió texto en vez de número
    Console.WriteLine("⚠️ Ingrese un número válido");
}
finally
{
    // Siempre se ejecuta — ideal para cerrar recursos
    Console.WriteLine("Proceso finalizado.");
}

🔒 Common Type System (CTS)

Define cómo se declaran y usan los tipos en .NET. Garantiza que int en C# y Integer en VB.NET sean el mismo tipo subyacente.

C# — Tipos de valor vs referencia
// TIPO DE VALOR: almacenado en el STACK — se COPIA al asignar
int stockOriginal = 50;
int stockCopia = stockOriginal; // copia independiente
stockCopia = 30;
Console.WriteLine(stockOriginal); // 50 — no cambió

// TIPO DE REFERENCIA: almacenado en el HEAP — comparte dirección
Producto p1 = new Producto("Polera", 9990m);
Producto p2 = p1; // ambos apuntan al mismo objeto
p2.Precio = 12990m;
Console.WriteLine(p1.Precio); // 12990 — sí cambió

🧵 Thread Manager

Gestiona los hilos de ejecución. El CLR provee el Thread Pool — conjunto de hilos reutilizables para operaciones paralelas.

🏬 En Almacenes Por Ahí: Cuando la app web atiende 50 cajeros simultáneamente, el Thread Manager asigna un hilo a cada petición sin crear uno nuevo para cada cliente.

📦 Paso 3: CIL — Common Intermediate Language
📐 Definición técnica
Lenguaje de bajo nivel e independiente de la plataforma al que se compilan TODOS los lenguajes .NET. Requiere que el CLR lo convierta mediante JIT para ejecutarse.
🎵 Analogía de la partitura musical
Tu código C# = la canción que compones. El CIL = la partitura. Es independiente del instrumento. El JIT es el músico que la toca en el procesador específico. La misma partitura (CIL) corre en Windows, Linux y macOS.
📥 Paso 4: Class Loader del CLR

Al presionar F5: carga el .dll desde disco a RAM, verifica la integridad del ensamblado, carga dependencias (System.dll, etc.) y prepara los metadatos para el JIT.

⚡ Paso 5: JIT — Just-In-Time Compiler

Primera vez que se llama un método → lo compila a código nativo del CPU actual → cachea en memoria. Llamadas siguientes usan el código nativo sin recompilar.

📐Anatomía de una Clase C#
❓ ¿Por qué usamos _nombre con guion bajo?

En C# existe una convención ampliamente adoptada en la industria: los campos privados (las variables internas de la clase) se escriben con un guion bajo como prefijo (_nombre, _precio, _stock).

✅ El motivo técnico
Cuando tienes un campo _nombre y una propiedad pública Nombre, el compilador y tú saben exactamente cuál es cuál. Sin el guion bajo, podrías confundir el campo privado con la propiedad:

nombre = nombre — ¿cuál es el campo y cuál el parámetro?
_nombre = nombre — ¡sin ambigüedad!
💡 La analogía
Imagina el cajero de Almacenes Por Ahí. El campo privado _caja es la caja fuerte detrás del mesón (nadie la toca directamente). La propiedad pública Caja es la ventanilla de cobro (acceso controlado). El _ es la señal visual de "esto es interno — no tocar desde afuera".
C# — Por qué el _ evita errores reales
// ❌ SIN convenio — el compilador se confunde
public class Producto
{
    private string nombre;      // campo privado

    public Producto(string nombre)  // parámetro del constructor
    {
        nombre = nombre;            // ⚠️ ERROR LÓGICO: asigna el parámetro a sí mismo
                                    // el campo 'nombre' NUNCA se inicializa
    }
}

// ✅ CON convenio _ — cero ambigüedad
public class Producto
{
    private string _nombre;     // campo privado — prefijo _ lo identifica

    public Producto(string nombre) // parámetro sin _
    {
        _nombre = nombre;           // ✅ CORRECTO: campo recibe el valor del parámetro
    }
}

// ✅ ALTERNATIVA MÁS USADA HOY: propiedades auto-implementadas
// Cuando usas get; set; NO necesitas campo privado ni _
public class Producto
{
    // Sin campo explícito, sin _ — el compilador genera el campo interno
    public string  Nombre { get; set; }  // ← preferida en la práctica diaria
    public decimal Precio { get; set; }
    public int     Stock  { get; set; }

    public Producto(string nombre, decimal precio)
    {
        Nombre = nombre;  // propiedad (con mayúscula) — no hay confusión
        Precio = precio;
    }
}

namespace AlmacenesPA.Modelos
{
    // CLASE: plantilla para crear objetos
    // public = accesible desde cualquier namespace
    public class Producto
    {
        // =============================================
        // 1. CAMPOS (fields) — almacenan el estado
        //    Convención C#: privados con _ prefijo
        // =============================================
        private int     _id;
        private string  _nombre;
        private decimal _precio;
        private int     _stock;

        // =============================================
        // 2. PROPIEDADES — acceso controlado a campos
        //    C# usa get/set (diferencia clave con Java)
        // =============================================
        public int Id
        {
            get { return _id; }       // leer el campo privado
            set { _id = value; }     // escribir (value = dato asignado)
        }

        // Propiedad auto-implementada (versión corta más usada)
        public string  Nombre { get; set; }
        public decimal Precio { get; set; }

        // Propiedad con validación en el setter
        public int Stock
        {
            get => _stock;                // expresión lambda (forma corta)
            set
            {
                // Validar que el stock no sea negativo
                if (value < 0)
                    throw new ArgumentException("Stock no puede ser negativo");
                _stock = value;
            }
        }

        // Propiedad de solo lectura (calculada, sin setter)
        public bool HayStock => _stock > 0;

        // =============================================
        // 3. CONSTRUCTORES — inicializan el objeto
        //    Mismo nombre que la clase, sin tipo retorno
        // =============================================

        // Constructor por defecto (sin parámetros)
        public Producto()
        {
            Nombre = "Sin nombre";
            Precio = 0m;
            _stock = 0;
        }

        // Constructor con parámetros (el más usado)
        public Producto(string nombre, decimal precio, int stock)
        {
            // Usamos las propiedades (no campos) para activar validaciones
            Nombre = nombre;
            Precio = precio;
            Stock  = stock; // activa el setter con validación
        }

        // =============================================
        // 4. MÉTODOS — comportamiento del objeto
        // =============================================

        // Método de instancia: actúa sobre los datos del objeto
        public decimal PrecioConDescuento(int porcentaje)
        {
            if (porcentaje < 0 || porcentaje > 100)
                throw new ArgumentException("Porcentaje entre 0 y 100");

            return Precio * (1 - porcentaje / 100m);
        }

        // Método estático: pertenece a la CLASE, no a un objeto
        public static Producto CrearVacio()
        {
            return new Producto(); // usa el constructor por defecto
        }

        // Override de ToString() — representación textual del objeto
        public override string ToString()
        {
            return $"[{Id:D3}] {Nombre,-20} ${Precio,8:N0}  Stock:{Stock}";
        }
    }
}

// CLASE BASE abstracta — no se puede instanciar directamente
public abstract class PersonaBase
{
    public string Nombre { get; set; }
    public string RUT    { get; set; }

    // Método abstracto: OBLIGA a las subclases a implementarlo
    public abstract string ObtenerRol();

    // Método virtual: las subclases PUEDEN sobreescribirlo
    public virtual string Presentarse()
        => $"Hola, soy {Nombre} ({RUT})";
}

// SUBCLASE Cliente: hereda de PersonaBase
// En Java: class Cliente extends PersonaBase
// En C#:   class Cliente : PersonaBase   ← dos puntos
public class Cliente : PersonaBase
{
    public DateTime FechaRegistro { get; set; }

    public Cliente(string nombre, string rut)
    {
        Nombre = nombre;        // hereda de PersonaBase
        RUT    = rut;
        FechaRegistro = DateTime.Now;
    }

    // Implementación obligatoria del método abstracto
    public override string ObtenerRol() => "Cliente";

    // Sobreescritura del método virtual
    public override string Presentarse()
        => $"Cliente: {Nombre}";
}

// INTERFAZ — contrato que la clase debe cumplir
// En Java: interface ICalculable
// En C#:   interface ICalculable  ← convención: 'I' al inicio
public interface ICalculable
{
    decimal CalcularTotal();        // solo firma, sin cuerpo
    decimal AplicarDescuento(int pct);
}

// Clase implementa la interfaz
public class Venta : ICalculable
{
    public List<DetalleVenta> Detalles { get; set; } = new();

    public decimal CalcularTotal()
    {
        decimal total = 0;
        foreach (var d in Detalles)
            total += d.Cantidad * d.PrecioUnitario; // acumular
        return total;
    }

    public decimal AplicarDescuento(int pct)
        => CalcularTotal() * (1 - pct / 100m);
}

    public decimal PrecioConDescuento(int pct)
    {
        if (pct < 0 || pct > 100)
            throw new ArgumentException("Porcentaje entre 0 y 100");
        return Precio * (1 - pct / 100m); // precio reducido
    }

    public void ReducirStock(int cant)
    {
        if (cant > Stock)
            throw new InvalidOperationException($"Stock insuf. Disp: {Stock}");
        Stock -= cant; // equivale a: Stock = Stock - cant
    }

    // Override de ToString() — representación del objeto como texto
    public override string ToString()
    {
        string st = TieneStock() ? "✅" : "⛔";
        return $"[{Id:D3}] {Nombre,-22} ${Precio,8:N0}  {st}";
    }
}

using System.Collections.Generic; // necesario para List<T>

List<Producto> inventario = new List<Producto>();

Leer datos del usuario y validar con TryParse. Flujo base del módulo de ingreso de inventario.
C# .NET — Ingreso y validación con TryParse
// Console.Write sin salto de línea — espera en la misma línea
Console.Write("Nombre del producto: ");
string nombre = (Console.ReadLine() ?? "").Trim(); // Trim() elimina espacios

if (nombre == "")
{
    Console.WriteLine("Error: el nombre no puede estar vacío.");
    return; // salir del programa
}

// TryParse: intenta convertir sin lanzar excepción
// Si tiene éxito: retorna true y guarda el valor en 'out precio'
// Si falla (ej: usuario escribe "abc"): retorna false
// Mucho más seguro que decimal.Parse() que lanza excepción
Console.Write("Precio ($): ");
decimal precio;
if (!decimal.TryParse(Console.ReadLine(), out precio) || precio <= 0)
{
    Console.WriteLine("Error: precio inválido.");
    return;
}

// Lo mismo para el stock (int)
Console.Write("Stock: ");
int stock;
if (!int.TryParse(Console.ReadLine(), out stock) || stock < 0)
{
    Console.WriteLine("Error: stock inválido.");
    return;
}

// Validar talla contra lista permitida
string[] tallasOk = { "XS", "S", "M", "L", "XL" };
Console.Write($"Talla ({string.Join("/", tallasOk)}): ");
string talla = (Console.ReadLine() ?? "").ToUpper(); // ToUpper(): mayúscula

if (Array.IndexOf(tallasOk, talla) == -1)
{
    Console.WriteLine("Error: talla inválida.");
    return;
}

// Crear el producto con inicializador de objeto { } (alternativa al constructor)
Producto nuevo = new Producto
{
    Id     = 1,
    Nombre = nombre,
    Precio = precio,
    Stock  = stock,
    Talla  = talla
};

Console.WriteLine($"\n✅ Producto creado: {nuevo}");
Console.ReadKey();

📖 Glosario Técnico

Busca cualquier término. +2 XP por cada término explorado.

CLR — Common Language Runtime.NET

📐 Técnica: Motor de ejecución de .NET. Provee JIT, GC, seguridad de tipos y manejo de excepciones.

💬 Simple: El gerente de operaciones de tu programa. Equivale a la JVM de Java.

🏬 Almacenes: Con 50 cajeros simultáneos, el CLR garantiza memoria aislada para cada proceso.

CIL — Common Intermediate LanguageArquitectura

📐 Técnica: Bytecode independiente de plataforma al que compilan todos los lenguajes .NET. El CLR lo convierte a nativo via JIT.

💬 Simple: La partitura musical: independiente del instrumento. El mismo .dll corre en Windows, Linux y macOS.

JIT — Just-In-Time CompilerArquitectura

📐 Técnica: Compila CIL a código nativo del CPU en la primera invocación. Cachea para llamadas posteriores.

💬 Simple: Intérprete simultáneo: traduce al idioma del CPU mientras el programa corre.

FCL / BCL — Framework & Base Class Library.NET

📐 Técnica: BCL = clases base en cualquier .NET. FCL = BCL + APIs específicas de plataforma.

💬 Simple: La BCL es el pasillo básico del supermercado, la FCL es el supermercado completo.

decimal vs double — Tipos numéricosC#

📐 Técnica: decimal: 128 bits, exacto, sufijo m. double: 64 bits IEEE, puede tener errores de precisión en operaciones financieras.

⚠️ Regla: Siempre usa decimal para precios. 9990.50m es exacto. 9990.50 (double) puede dar 9990.499999...

namespace — Espacio de nombresC#

📐 Técnica: Contenedor lógico que agrupa tipos relacionados. Equivale al package de Java.

🏬 Ejemplo: namespace AlmacenesPA.Modelos agrupa Producto, Cliente, Venta.

Property — Propiedades (get/set)C#

📐 Técnica: Miembro de clase que encapsula acceso a un campo con lógica opcional.

⚖️ Java: Java: getNombre() / setNombre(). C#: propiedad Nombre con get; set; integrado.

Breakpoint — Punto de interrupciónVisual Studio

📐 Técnica: Marca en línea de código que pausa la ejecución para inspeccionar variables.

💬 Simple: Un "STOP" en la carretera del programa. Se pone con F9, se avanza con F10.

JVM vs CLR — EquivalenciaPrevio

📐 Técnica: JVM y CLR son máquinas virtuales que ejecutan bytecode/CIL, tienen JIT y GC generacional.

Diferencia clave: CLR fue diseñado para múltiples lenguajes desde el origen. JVM nació para Java y adoptó otros (Kotlin, Scala) después.

GC — Garbage Collector.NET

📐 Técnica: Subsistema del CLR con algoritmo generacional (Gen 0/1/2). Libera automáticamente objetos sin referencias activas.

💬 Simple: El equipo de aseo nocturno: limpia lo que ya no sirve sin interrumpir el trabajo del día.
