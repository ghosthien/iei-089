🔤 Interpolación de Strings

Antes de entrar a las colecciones, dominamos esto — lo usaremos en todos los ejemplos de la clase.

📐 Definición técnica
La interpolación de strings ($"...") es un mecanismo del compilador C# que permite incrustar expresiones dentro de un literal de string. El compilador de Roslyn la transforma en una llamada a string.Format() o StringBuilder optimizada. Disponible desde C# 6.0.
💡 En palabras simples
En vez de unir pedazos de texto con el operador + (concatenar), escribes el string completo y metes las variables directamente dentro de {llaves}. El código queda mucho más legible — y el compilador lo optimiza automáticamente.

❌ Concatenación vs ✅ Interpolación

❌ CONCATENACIÓN — difícil de leer y propensa a errores
// Unir pedazos con + es tedioso y confuso
string msg = "Producto: " + nombre
+ " | Precio: $" + precio
+ " | Stock: " + stock
+ " unidades";
// Si olvidas un + o una comilla → error
✅ INTERPOLACIÓN — limpio, directo y el compilador lo optimiza
// El $ activa la interpolación
// Las {llaves} insertan la variable
string msg = $"Producto: {nombre}
| Precio: ${precio:N0}
| Stock: {stock} unidades";
// Mismo resultado, mucho más claro
C# — Interpolación completa con formatos 
// La $ antes de la comilla activa la interpolación
// Dentro de {llaves} puedes poner cualquier expresión C#

string  nombre  = "Polera básica";
decimal precio  = 9990.50m;
int     stock   = 120;
bool    activo  = true;
DateTime fecha  = DateTime.Now;

// Interpolación básica: solo meter la variable
Console.WriteLine($"Nombre: {nombre}");

// Con formato numérico:  :N0 = número con separador de miles, sin decimales
Console.WriteLine($"Precio: ${precio:N0}");          // Precio: $9.991
Console.WriteLine($"Precio: ${precio:F2}");          // Precio: $9990,50

// Con alineación: {valor,-20} = alinear izquierda en 20 chars
Console.WriteLine($"{nombre,-25} ${precio,10:N0}"); // columnas bien alineadas

// Con expresión directa dentro de las llaves
Console.WriteLine($"¿Hay stock? {(stock > 0 ? "Sí" : "No")}");

// Con formato de fecha
Console.WriteLine($"Fecha: {fecha:dd/MM/yyyy HH:mm}"); // 19/03/2025 14:30

// Con D3: muestra el número con mínimo 3 dígitos (rellena con ceros)
Console.WriteLine($"ID: {42:D3}");  // ID: 042

// Interpolación multilínea (raw string — C# 11+)
string ficha = $"""
    ╔══════════════════════════╗
    ║  {nombre,-24}║
    ║  Precio: ${precio,14:N0}  ║
    ║  Stock:  {stock,14} uds  ║
    ╚══════════════════════════╝
    """;
Console.WriteLine(ficha);
🔲Cómo hacer los caracteres de caja — ╔══════════╗
¿Qué son estos caracteres?
Son caracteres especiales llamados caracteres de dibujo de caja (Box Drawing Characters), parte del estándar Unicode. No son imágenes — son letras como la "a" o la "b", pero con forma de línea. En C# se escriben directamente dentro de los strings porque Visual Studio y la consola moderna soportan Unicode.
¿Cómo los escribo en Visual Studio?
Cópialos directamente desde aquí: ╔ ╗ ╚ ╝ ║ ═ ╠ ╣ ╦ ╩ ╬

O en Windows: Win+. abre el panel de emojis y caracteres especiales. En macOS: Ctrl+Cmd+Space. También puedes usar el Mapa de caracteres de Windows.
📋 Tabla de caracteres más usados en consola
Esquinas: ╔ (arriba-izq)   ╗ (arriba-der)   ╚ (abajo-izq)   ╝ (abajo-der)
Líneas: ═ (horizontal doble)   ║ (vertical doble)   ─ (horizontal simple)   │ (vertical simple)
Cruces: ╠ (T izquierda)   ╣ (T derecha)   ╦ (T abajo)   ╩ (T arriba)   ╬ (cruz)
Otros útiles: • (bullet)   →   ←   ↑   ↓   ✅   ❌   ⚠   🔔   🎮   🛒   🍕
C# — Cómo dibujar cuadros en la consola (3 técnicas)
// ─────────────────────────────────────────────────────────
// TÉCNICA 1: Copiar y pegar los caracteres directamente
// La más simple — los caracteres van dentro del string normal
// ─────────────────────────────────────────────────────────
Console.WriteLine("╔══════════════════════╗");
Console.WriteLine("║  TIENDA GAMER v1.0  ║");
Console.WriteLine("╚══════════════════════╝");

// ─────────────────────────────────────────────────────────
// TÉCNICA 2: Usar new string(char, cantidad) para líneas largas
// Más flexible — puedes cambiar el ancho con una variable
// new string('═', 30) → genera 30 caracteres '═' seguidos
// ─────────────────────────────────────────────────────────
int ancho = 30;
Console.WriteLine($"╔{new string('═', ancho)}╗"); // ╔══════════════════════════════╗
Console.WriteLine($"║{"SUPERMERCADO",ancho/2+6}{"",ancho/2-6}║");
Console.WriteLine($"╚{new string('═', ancho)}╝"); // ╚══════════════════════════════╝

// ─────────────────────────────────────────────────────────
// TÉCNICA 3: Cuadro con datos variables (la más profesional)
// Usar interpolación con alineación para que el contenido encaje
// ─────────────────────────────────────────────────────────
string nombreJuego = "FIFA 25";
decimal precio     = 59990m;
int     stock      = 12;

Console.ForegroundColor = ConsoleColor.Cyan;
Console.WriteLine("╔══════════════════════════════════╗");
Console.WriteLine($"║  {nombreJuego,-32}║"); // {var,-32} alinea a la izq en 32 chars
Console.WriteLine("╠══════════════════════════════════╣");
Console.WriteLine($"║  Precio: ${precio,21:N0}  ║");      // alineado a la derecha
Console.WriteLine($"║  Stock:  {stock,22} uds  ║");
Console.WriteLine("╚══════════════════════════════════╝");
Console.ResetColor(); // SIEMPRE restaurar el color después

// ─────────────────────────────────────────────────────────
// TÉCNICA 4: Separadores simples con new string
// Más limpio que copiar ─────────────────── manualmente
// ─────────────────────────────────────────────────────────
Console.WriteLine(new string('═', 40)); // ════════════════════════════════════════
Console.WriteLine(new string('─', 40)); // ────────────────────────────────────────
Console.ReadKey();
💡 Consejo — alineación para que los ║ queden rectos
El truco para que las líneas verticales ║ queden alineadas es usar interpolación con un ancho fijo: {variable,-N} donde N es el número de caracteres que ocupa esa parte del cuadro. Si el texto es más corto, se rellena con espacios automáticamente. Prueba distintos valores de N hasta que el cuadro quede recto.

📦 Arrays (Arreglos)

Estructura de datos de tamaño fijo. La base de todas las colecciones. Namespace: System (BCL base — siempre disponible, sin using extra).

📦 Array — Colección de tamaño fijo BCL · System
📐 Definición técnica
Estructura de datos lineal que almacena elementos del mismo tipo en posiciones contiguas de memoria. El tamaño se fija en la creación y no puede cambiar. Acceso en tiempo O(1) mediante índice. Los índices comienzan en 0. Clase base System.Array de la BCL.
💡 Analogía profesional
El array es como una estantería con un número fijo de casilleros. Cada casillero tiene un número (índice) y puedes ir directamente al casillero 47 sin revisar los anteriores. Perfecto cuando sabes exactamente cuántos elementos tendrás.
🔧 ¿Cuándo usar un Array?
✅ Tamaño fijo y conocido desde el inicio (5 géneros, 7 días, 12 meses) · ✅ Cuando necesitas acceder a elementos por posición numérica frecuentemente · ✅ Para tablas con filas y columnas fijas (array 2D) · ❌ NO usar si la cantidad de elementos puede cambiar — ahí va List<T>.
📦 Sintaxis esencial del Array
tipo[] nombre = new tipo[tamaño]; → declarar vacío
tipo[] nombre = { val1, val2, val3 }; → declarar con valores
nombre[0] → primer elemento (índice 0)
nombre[nombre.Length - 1] → último elemento
nombre.Length → cantidad de elementos

🎮 Tienda de Videojuegos — La tienda vende 5 géneros fijos (nunca cambian). Usamos un array porque el tamaño es fijo y conocido.

¿Por qué array aquí y no List?
Los géneros de videojuegos son fijos: Acción, RPG, Deportes, Terror, Aventura — siempre serán 5. El array es perfecto para datos que nunca cambian de cantidad. La List se usa cuando no sabes cuántos elementos habrá.
C# — Array básico: géneros de videojuegosBásico
// DECLARAR un array: tipo[] nombre = new tipo[tamaño];
// El tamaño se define UNA VEZ y no cambia
decimal[] precios = new decimal[5]; // array de 5 decimales (todos en 0)

// ASIGNAR valores por índice (siempre empieza en 0)
precios[0] = 9990m;   // primer elemento  → índice 0
precios[1] = 24990m;  // segundo elemento → índice 1
precios[2] = 4990m;   // tercer elemento  → índice 2
precios[3] = 49990m;  // cuarto elemento  → índice 3
precios[4] = 12990m;  // quinto elemento  → índice 4 (último)

// INICIALIZAR directamente con valores (forma corta)
string[] tallas = { "XS", "S", "M", "L", "XL", "XXL" };

// LEER un elemento específico
Console.WriteLine($"Primer precio: ${precios[0]:N0}");  // $9.990
Console.WriteLine($"Última talla:  {tallas[5]}");       // XXL

// RECORRER con foreach — la forma más limpia
// foreach no necesita índice, recorre todos los elementos
Console.WriteLine("\nLista de precios:");
foreach (decimal precio in precios)
{
    Console.WriteLine($"  ${precio:N0}");
}

// RECORRER con for — cuando necesitas el índice
Console.WriteLine("\nTallas disponibles:");
for (int i = 0; i < tallas.Length; i++) // .Length = número de elementos
{
    Console.WriteLine($"  [{i}] {tallas[i]}");
}

🛒 Supermercado — Necesitamos una tabla de stock: 3 secciones × 4 productos. Un array 2D es como una hoja de cálculo con filas y columnas.

¿Cuándo usar un array 2D?
Cuando los datos son una tabla rectangular con filas y columnas fijas. Ejemplo: 3 secciones del supermercado × 4 productos. Accedes con [fila, columna]. GetLength(0) = número de filas. GetLength(1) = número de columnas.
C# — Array 2D: stock supermercado por secciónIntermedio
// ARRAY BIDIMENSIONAL: [filas, columnas]
// Útil para tablas de datos (como una hoja de cálculo)
// Aquí: 3 secciones × 4 productos = tabla de stock del supermercado
int[,] stock = new int[3, 4]  // 3 secciones × 4 productos
{
    { 120, 45, 80, 200 },   // sucursal 0 (Santiago Centro)
    {  60, 30, 90,  50 },   // sucursal 1 (Providencia)
    {  40, 70, 20, 110 },   // sucursal 2 (Ñuñoa)
};

string[] secciones = { "Abarrotes", "Panadería", "Lácteos" };
string[] productos  = { "Polera", "Jeans", "Chaqueta", "Calcetines" };

// RECORRER array 2D con dos for anidados
Console.WriteLine("=== STOCK POR SUCURSAL ===");
Console.Write($"{"",14}"); // espacio para los nombres de columna
foreach (string p in productos)
    Console.Write($"{p,12}");  // encabezado de columnas alineado
Console.WriteLine();

for (int s = 0; s < stock.GetLength(0); s++)  // .GetLength(0) = filas
{
    Console.Write($"{secciones[s],-14}");
    for (int p = 0; p < stock.GetLength(1); p++) // .GetLength(1) = columnas
    {
        Console.Write($"{stock[s,p],12}");  // alinear a derecha en 12 chars
    }
    Console.WriteLine();
}

// OPERACIONES ÚTILES de System.Array
int[] stocks1D = { 120, 45, 80, 200, 60 };

Array.Sort(stocks1D);      // ordena de menor a mayor
Array.Reverse(stocks1D);   // invierte el orden (ahora de mayor a menor)

Console.WriteLine($"\nMáximo stock:  {stocks1D[0]}");   // primer tras reverse
Console.WriteLine($"Mínimo stock:  {stocks1D[^1]}");   // ^1 = último elemento
Console.WriteLine($"Total filas:   {stock.GetLength(0)}");
Console.WriteLine($"Total columnas:{stock.GetLength(1)}");

🍕 Pizzería Don Corleone — Informe de ventas semanal. Detecta el día de mayor venta y calcula el total de pizzas vendidas en la semana.

C# — Array 2D profesional: ventas semanales Pizzería Don CorleoneProfesional
// Tabla: 7 días (filas) × 3 tipos de pizza (columnas)
// Objetivo: el día más exitoso y la pizza más vendida

string[] dias   = { "Lunes","Martes","Miércoles","Jueves","Viernes","Sábado","Domingo" };
string[] pizzas = { "Margarita", "4 Quesos", "Napolitana" };

// Unidades vendidas: ventas[dia, pizza]
int[,] ventas = {
    { 12,  8,  5 }, // Lunes
    {  9,  6,  4 }, // Martes
    { 14, 10,  7 }, // Miércoles
    { 11,  9,  6 }, // Jueves
    { 25, 18, 12 }, // Viernes
    { 40, 30, 20 }, // Sábado ← mejor día
    { 35, 22, 15 }, // Domingo
};

// Encabezado de la tabla con interpolación y alineación
Console.WriteLine("=== PIZZERÍA DON CORLEONE — VENTAS SEMANALES ===");
Console.Write($"{"DÍA",-12}");
foreach (string p in pizzas) Console.Write($"{p,12}");
Console.WriteLine($"{"TOTAL",8}");
Console.WriteLine(new string('─', 50));

int mejorDiaIdx = 0, maxDia = 0, totalSemana = 0;

for (int d = 0; d < dias.Length; d++)
{
    int totalDia = 0;
    Console.Write($"{dias[d],-12}");
    for (int p = 0; p < pizzas.Length; p++)
    {
        Console.Write($"{ventas[d,p],12}");
        totalDia += ventas[d, p];
    }
    Console.WriteLine($"{totalDia,8}");
    totalSemana += totalDia;
    if (totalDia > maxDia) { maxDia = totalDia; mejorDiaIdx = d; }
}

Console.WriteLine(new string('─', 50));
Console.WriteLine($"Total semana: {totalSemana} pizzas");
Console.WriteLine($"Mejor día:    {dias[mejorDiaIdx]} con {maxDia} pizzas");
Console.ReadKey();

📋 List<T> — La Colección Estrella

Colección dinámica de la BCL. La más usada en C# del día a día. Namespace: System.Collections.Generic.

📋 List<T> — Colección dinámica tipada BCL · System.Collections.Generic
📐 Definición técnica
List<T> implementa IList<T> e ICollection<T>. Internamente usa un array que se redimensiona automáticamente (duplica capacidad al superar el límite). Acceso aleatorio O(1), inserción al final O(1) amortizado, búsqueda lineal O(n).
💡 En palabras simples
Como un array pero que crece o se encoge solo. No necesitas saber de antemano cuántos elementos tendrás. Es el ArrayList tipado de Java: no admite tipos incorrectos y no necesitas hacer cast.
🔧 Import necesario + Sintaxis esencial
using System.Collections.Generic; — en .NET 8 ya está incluido automáticamente.

List<string> lista = new(); → lista vacía
lista.Add("valor"); → agregar al final
lista[0] → acceder por índice
lista.Count → cuántos elementos tiene
lista.Remove("valor"); → eliminar por valor
lista.Contains("valor"); → ¿existe el valor?
lista.Find(x => condición); → buscar el primero que cumple
lista.FindAll(x => condición); → buscar todos los que cumplen

🎮 Tienda de Videojuegos — La tienda tiene un catálogo de juegos que varía: llegan juegos nuevos y se retiran otros. Usamos List porque el tamaño cambia.

¿Por qué List y no array aquí?
A diferencia del array, la List<T> puede crecer o reducirse en cualquier momento. Si llega un juego nuevo: Add(). Si se retira: Remove(). No necesitas saber de antemano cuántos juegos habrá.
C# — List básico: catálogo de videojuegosBásico
// List<string> → lista dinámica de strings
// A diferencia del array, NO necesitas decir cuántos habrá
// using System.Collections.Generic; (en .NET 8 ya está incluido)
List<string> catalogo = new List<string>();

// Add(): agregar un elemento al FINAL de la lista
catalogo.Add("FIFA 25");
catalogo.Add("Elden Ring");
catalogo.Add("Spider-Man 2");
catalogo.Add("Halo Infinite");

// Count → cuántos elementos tiene la lista (como .Length del array)
Console.WriteLine($"Juegos en catálogo: {catalogo.Count}"); // 4

// Acceder por índice (igual que array)
Console.WriteLine($"Primer juego: {catalogo[0]}"); // FIFA 25

// Recorrer con foreach
Console.WriteLine("
Catálogo completo:");
foreach (string juego in catalogo)
    Console.WriteLine($"  • {juego}");

// Remove(): eliminar un elemento por valor
// La lista se ajusta automáticamente — no quedan "huecos"
catalogo.Remove("Halo Infinite");
Console.WriteLine($"
Tras eliminar Halo: {catalogo.Count} juegos"); // 3

// Contains(): verificar si un elemento está en la lista
bool tieneElden = catalogo.Contains("Elden Ring");
Console.WriteLine($"¿Tiene Elden Ring? {tieneElden}"); // True

// Insert(): agregar en una posición específica
// Insert(1, ...) → insertar en la posición 1, empujando los demás
catalogo.Insert(1, "God of War"); // queda entre FIFA y Elden Ring
Console.WriteLine($"
Tras insertar God of War en pos.1: {catalogo[1]}");
Console.ReadKey();

🛒 Supermercado — El supermercado tiene una lista de productos con nombre, precio y stock. Necesita buscar productos y actualizar stocks.

¿Qué es una lambda (p => ...)?
Una lambda es una mini-función que se escribe en una sola línea. p => p.Precio > 1000 significa: "para cada p de la lista, evalúa si p.Precio es mayor a 1000". El => se lee como "tal que". Se usa en Find(), FindAll(), Sort() para decirle cómo buscar u ordenar.
C# — List con objetos: productos del supermercadoIntermedio
// List<T> con objetos propios — no solo strings
// Cada elemento de la lista es un objeto Producto
var productos = new List<Producto>();

// Agregar objetos completos (usando record para simplificar)
productos.Add(new Producto(1, "Arroz 1kg",      1190m, 80));
productos.Add(new Producto(2, "Leche 1L",       890m,  120));
productos.Add(new Producto(3, "Pan Molde",      1490m, 0));  // agotado
productos.Add(new Producto(4, "Aceite 1L",      2990m, 45));
productos.Add(new Producto(5, "Pasta Spaghetti", 890m, 60));

// Find(): busca y retorna el PRIMERO que cumple la condición
// p => p.Id == 2 → "busca el producto donde Id sea igual a 2"
// Si no encuentra nada, retorna null (por eso usamos Producto?)
Producto? leche = productos.Find(p => p.Id == 2);
if (leche != null)
    Console.WriteLine($"Encontrado: {leche.Nombre} — ${leche.Precio:N0}");

// FindAll(): retorna una NUEVA lista con TODOS los que cumplen la condición
// Aquí buscamos todos los productos con stock mayor a 0
List<Producto> conStock = productos.FindAll(p => p.Stock > 0);
Console.WriteLine($"Productos disponibles: {conStock.Count}"); // 4 (el Pan tiene 0)

// Sort(): ordena la lista según el criterio que definas
// (a, b) => a.Precio.CompareTo(b.Precio) → ordenar de menor a mayor precio
// CompareTo retorna: negativo si a < b, 0 si iguales, positivo si a > b
productos.Sort((a, b) => a.Precio.CompareTo(b.Precio));
Console.WriteLine("
=== PRODUCTOS ORDENADOS POR PRECIO ===");
foreach (var p in productos)
    Console.WriteLine($"  [{p.Id:D2}] {p.Nombre,-20} ${p.Precio:N0,8}  Stock:{p.Stock}");

// RemoveAll(): elimina TODOS los que cumplen la condición y retorna cuántos eliminó
int eliminados = productos.RemoveAll(p => p.Stock == 0); // quitar los agotados
Console.WriteLine($"
Productos agotados eliminados: {eliminados}");
Console.WriteLine($"Quedan en lista: {productos.Count}");
Console.ReadKey();

// La clase Producto (simplificada con record)
record Producto(int Id, string Nombre, decimal Precio, int Stock);

🍕 Pizzería Don Corleone — Sistema de pedidos con clase de servicio. Separa la lógica del negocio de la presentación.

// En proyectos reales la lógica de la lista va en una CLASE SEPARADA
// Esto se llama "Servicio" o "Repositorio" — patrón muy usado en .NET
public class PizzaService
{
    // _pedidos: campo privado — el guión bajo (_) indica campo privado
    // readonly: la referencia no cambia; la lista sí puede crecer
    // new(): forma corta de new List<Pedido>()
    private readonly List<Pedido> _pedidos = new();
    private int _proximoId = 1; // auto-incremental para los IDs

    // Método público para agregar un pedido con validación
    public Pedido AgregarPedido(string cliente, string pizza, int cantidad)
    {
        // Validar antes de agregar
        if (cantidad <= 0) throw new ArgumentException("Cantidad debe ser mayor a 0");

        var pedido = new Pedido(_proximoId++, cliente, pizza, cantidad);
        _pedidos.Add(pedido);  // agregar al final de la lista
        return pedido;          // retornar el pedido creado
    }

    // Buscar pedidos de un cliente específico (puede haber varios)
    public List<Pedido> BuscarPorCliente(string cliente)
        => _pedidos.FindAll(p => p.Cliente.Contains(cliente,
               StringComparison.OrdinalIgnoreCase)); // ignora mayúsculas

    // Informe: muestra todos los pedidos y el total
    public void MostrarInforme()
    {
        Console.WriteLine("╔══════════════════════════════════════╗");
        Console.WriteLine("║  PIZZERÍA DON CORLEONE — PEDIDOS    ║");
        Console.WriteLine("╠══════════════════════════════════════╣");
        // Mostrar cada pedido de la lista con foreach
        foreach (var p in _pedidos)
            Console.WriteLine($"║ #{p.Id:D3} {p.Cliente,-15} {p.Pizza,-12} x{p.Cantidad} ║");
        Console.WriteLine("╠══════════════════════════════════════╣");
        // Calcular total de pizzas vendidas
        int totalPizzas = 0;
        foreach (var p in _pedidos) totalPizzas += p.Cantidad;
        Console.WriteLine($"║ Total pedidos: {_pedidos.Count,-5} Pizzas: {totalPizzas,-9}║");
        Console.WriteLine("╚══════════════════════════════════════╝");
    }
}

🗂️ Dictionary<K,V> — Búsqueda por Clave

Colección de pares clave-valor. Acceso en tiempo O(1) por clave. Namespace: System.Collections.Generic.

🗂️ Dictionary<K,V> — Mapa de clave-valor BCL · System.Collections.Generic
📐 Definición técnica
Implementa IDictionary<K,V> usando una tabla hash internamente. Cada clave es única. Búsqueda, inserción y eliminación en tiempo O(1) promedio. El tipo K debe implementar GetHashCode() y Equals(). Equivale a HashMap de Java.
💡 En palabras simples
Como un diccionario real: buscas una palabra (clave) y obtienes su definición (valor) directo — sin leer todo el libro. Perfecto para: buscar un producto por ID, buscar un cliente por RUT, o el precio de un artículo por su código.
🔧 Sintaxis esencial del Dictionary
Dictionary<string, decimal> d = new(); → diccionario vacío
d["clave"] = valor; → agregar o actualizar
d["clave"] → leer (⚠ lanza excepción si no existe)
d.ContainsKey("clave") → verificar si existe (antes de leer)
d.TryGetValue("clave", out var v) → leer de forma segura ✅
d.Remove("clave"); → eliminar un par
d.Count → cuántos pares tiene
foreach (var (k,v) in d) → recorrer todos los pares

🎮 Tienda de Videojuegos — Catálogo de precios por código de juego. El cajero escanea el código y necesita el precio al instante.

¿Por qué Dictionary y no List aquí?
Con una List tendrías que recorrer todos los juegos hasta encontrar el que buscas (lento). Con un Dictionary, le dices directamente "dame el precio del código PS5-001" y lo entrega en tiempo O(1) — no importa si tienes 10 o 10.000 juegos.
C# — Dictionary básico: precios por código de juegoBásico
// Dictionary<K, V>: colección de pares clave-valor
// K (Key)   = tipo de la clave → string (el código del juego)
// V (Value) = tipo del valor   → decimal (el precio)
// Cada clave es ÚNICA — no pueden repetirse
Dictionary<string, decimal> precios = new()
{
    // clave      → valor
    { "PS5-001",  59990m }, // FIFA 25
    { "PS5-002",  49990m }, // Elden Ring
    { "XB-001",   39990m }, // Halo Infinite
    { "PC-001",   29990m }, // Minecraft
};

// ACCEDER por clave → directo, O(1), sin recorrer nada
Console.WriteLine($"Precio FIFA 25: ${precios["PS5-001"]:N0}"); // $59.990

// AGREGAR o ACTUALIZAR un par clave-valor
// Si la clave NO existe → la crea. Si YA existe → actualiza el valor
precios["SW-001"] = 44990m; // agregar Nintendo Switch game
precios["PS5-001"] = 54990m; // actualizar precio de FIFA 25

// ContainsKey(): verificar si una clave existe ANTES de acceder
// IMPORTANTE: si intentas acceder a una clave que no existe
// sin verificar, el programa lanza una excepción y se cae
string buscar = "NS-999";
if (precios.ContainsKey(buscar))
    Console.WriteLine($"{buscar}: ${precios[buscar]:N0}");
else
    Console.WriteLine($"El código '{buscar}' no existe en el catálogo");

// TryGetValue(): la forma MÁS SEGURA de buscar
// Retorna true si encontró la clave, y guarda el valor en 'precio'
// No lanza excepción si no existe — siempre usarla en producción
if (precios.TryGetValue("XB-001", out decimal precio))
    Console.WriteLine($"Halo Infinite: ${precio:N0}");  // $39.990

// RECORRER todos los pares con foreach
// Cada par es un KeyValuePair<string, decimal>
Console.WriteLine("
=== TODOS LOS PRECIOS ===");
foreach (KeyValuePair<string, decimal> par in precios)
    Console.WriteLine($"  {par.Key,-10} ${par.Value:N0}");

// Count → número de pares en el Dictionary
Console.WriteLine($"
Total juegos: {precios.Count}");
Console.ReadKey();

🛒 Supermercado — Contar cuántas veces se vendió cada producto en el día. Un Dictionary funciona perfecto como contador.

Dictionary como contador
Un patrón muy común: usar un Dictionary para contar frecuencias. La clave es el nombre del producto, el valor es cuántas veces apareció. Por cada venta: si la clave ya existe, incrementa el contador; si no, créala con valor 1.
C# — Dictionary intermedio: contador de ventas supermercadoIntermedio
// Simular el registro de ventas del supermercado durante el día
// Cada string es el nombre del producto que pasó por caja
string[] ventasDia = {
    "Leche", "Pan", "Arroz", "Leche", "Pan",
    "Arroz", "Leche", "Aceite", "Pan", "Leche",
    "Aceite", "Arroz", "Pasta", "Pan", "Leche"
};

// Dictionary para contar: clave=producto, valor=cantidad vendida
Dictionary<string, int> contador = new();

// Recorrer cada venta y contar
foreach (string producto in ventasDia)
{
    // Si el producto YA está en el Dictionary → incrementar su contador
    if (contador.ContainsKey(producto))
        contador[producto]++;     // ++ incrementa en 1
    else
        contador[producto] = 1;  // primera vez que aparece → iniciar en 1
}

// Mostrar el resumen de ventas
Console.WriteLine("=== RESUMEN DE VENTAS DEL DÍA ===");
foreach (var (producto, cantidad) in contador) // deconstrucción moderna
    Console.WriteLine($"  {producto,-12}: {cantidad} unidades");

// Encontrar el producto más vendido
// Empieza con el primero y va comparando con el resto
string masVendido = "";
int    maxCantidad = 0;
foreach (var (prod, cant) in contador)
{
    if (cant > maxCantidad)
    {
        maxCantidad = cant;
        masVendido  = prod;
    }
}
Console.WriteLine($"
🏆 Producto más vendido: {masVendido} con {maxCantidad} ventas");
Console.WriteLine($"Total ventas del día: {ventasDia.Length}");
Console.ReadKey();

C# — Dictionary profesional: punto de venta Pizzería Don CorleoneProfesional
// Sistema de caja — el cajero ingresa códigos y el Dictionary
// retorna precio y nombre al instante sin recorrer toda la lista
public class CajaPizzeria
{
    // Clave: código de pizza | Valor: tupla con nombre y precio
    // Una tupla agrupa varios valores sin necesidad de una clase separada
    private readonly Dictionary<string, (string Nombre, decimal Precio)> _catalogo;

    // Historial de ventas: código → cuántas veces se vendió hoy
    private readonly Dictionary<string, int> _ventasHoy = new();

    public CajaPizzeria()
    {
        // Inicializar el catálogo con valores por defecto
        _catalogo = new()
        {
            ["MAR"] = ("Margarita",  8990m),
            ["4QS"] = ("4 Quesos",   11990m),
            ["NAP"] = ("Napolitana", 9990m),
            ["PEP"] = ("Pepperoni",  10990m),
        };
    }

    // Registrar una venta: retorna el total de esa venta
    public decimal RegistrarVenta(string codigo, int cantidad)
    {
        // TryGetValue: buscar de forma segura sin excepción
        if (!_catalogo.TryGetValue(codigo, out var pizza))
        {
            Console.WriteLine($"⚠ Código '{codigo}' no encontrado");
            return 0;
        }

        decimal total = pizza.Precio * cantidad;
        Console.WriteLine($"✅ {pizza.Nombre} × {cantidad} = ${total:N0}");

        // Registrar en el historial (patrón contador)
        if (_ventasHoy.ContainsKey(codigo))
            _ventasHoy[codigo] += cantidad;
        else
            _ventasHoy[codigo] = cantidad;

        return total;
    }

    // Informe de cierre: resumen de ventas del día
    public void InformeCierre()
    {
        Console.WriteLine("
=== CIERRE DEL DÍA — PIZZERÍA DON CORLEONE ===");
        decimal totalDia = 0;
        foreach (var (codigo, cantidad) in _ventasHoy)
        {
            var pizza = _catalogo[codigo]; // seguro porque código viene del historial
            decimal subtotal = pizza.Precio * cantidad;
            Console.WriteLine($"  {pizza.Nombre,-15}: {cantidad,3} uds  ${subtotal:N0}");
            totalDia += subtotal;
        }
        Console.WriteLine($"  {"TOTAL DEL DÍA",-15}:       ${totalDia:N0}");
    }
}

// USAR el sistema de caja
var caja = new CajaPizzeria();
caja.RegistrarVenta("MAR", 2);  // 2 Margaritas
caja.RegistrarVenta("4QS", 1);  // 1 Pizza 4 Quesos
caja.RegistrarVenta("XYZ", 1);  // código inexistente → muestra advertencia
caja.RegistrarVenta("MAR", 3);  // 3 Margaritas más
caja.InformeCierre();
Console.ReadKey();

🔁 Queue<T> y Stack<T>

Dos colecciones con comportamientos opuestos. Namespace: System.Collections.Generic. Contextos: 🎮 Tienda Gamer (básico) · 🛒 Supermercado (intermedio) · 🍕 Pizzería Don Corleone (profesional).

🟡 Queue<T> — Cola (FIFO) BCL · System.Collections.Generic
📐 Definición técnica
First In, First Out (FIFO). Implementa ICollection<T>. Enqueue() agrega al final en O(1). Dequeue() extrae del frente en O(1). Peek() consulta sin extraer. Count retorna el tamaño.
💡 Explicación sin jerga
Es la fila del banco. El primero que llegó es el primero que atienden. Si llegas último, esperas hasta que pasen todos los anteriores. Enqueue() = ponerse al final de la fila. Dequeue() = el cajero llama al siguiente. Peek() = mirar quién es el próximo sin atenderlo.

🎮 Tienda Gamer — Los clientes hacen cola para reservar el nuevo lanzamiento. El primero que llegó es el primero que es atendido.

¿Qué es FIFO?
First In, First Out — El primero que entra es el primero que sale. Exactamente como la fila del banco o la cola de un videojuego en línea. Enqueue() = ponerse al final de la fila. Dequeue() = el cajero atiende al primero. Peek() = mirar quién es el siguiente sin atenderlo aún.
C# — Queue básico: reservas Tienda GamerBásico
// Queue<T> — El primero que se registró es el primero que recibe su juego
// Contexto: cola de reservas para el lanzamiento de FIFA 26
var colaReservas = new Queue<string>();

// Enqueue(): agrega al FINAL de la cola (se pone a esperar)
colaReservas.Enqueue("Carlos Pérez");   // llegó primero
colaReservas.Enqueue("Ana Torres");    // llegó segundo
colaReservas.Enqueue("Luis Rojas");    // llegó tercero

Console.WriteLine($"Clientes en cola: {colaReservas.Count}"); // 3

// Peek(): ver quién es el siguiente SIN sacarlo de la cola
Console.WriteLine($"Próximo a atender: {colaReservas.Peek()}"); // Carlos

// Dequeue(): atender al primero de la cola (FIFO)
// Cada llamada saca al cliente que lleva más tiempo esperando
while (colaReservas.Count > 0)
{
    string cliente = colaReservas.Dequeue(); // extrae el más antiguo
    Console.WriteLine($"✅ Entregando FIFA 26 a: {cliente}");
    Console.WriteLine($"   Clientes restantes: {colaReservas.Count}");
}
// Salida (orden de llegada - FIFO):
// ✅ Entregando FIFA 26 a: Carlos Pérez   ← llegó 1°, sale 1°
// ✅ Entregando FIFA 26 a: Ana Torres
// ✅ Entregando FIFA 26 a: Luis Rojas     ← llegó 3°, sale 3°
Console.ReadKey();

// Sistema real de pedidos con clase de servicio
// Encapsula la Queue dentro de una clase para organizar mejor el código
public class SistemaPedidos
{
    private readonly Queue<Pedido> _cola = new();
    private int _numeroPedido = 1;

    // Registrar pedido nuevo: llega al FINAL de la cola
    public void RegistrarPedido(string mesa, string pizza, int cantidad)
    {
        var p = new Pedido(_numeroPedido++, mesa, pizza, cantidad);
        _cola.Enqueue(p);
        Console.WriteLine($"📝 Pedido #{p.Id:D3} registrado — {mesa}: {cantidad}x {pizza}");
        Console.WriteLine($"   Pedidos en cola: {_cola.Count}");
    }

    // Procesar el siguiente pedido (el más antiguo — FIFO)
    public void ProcesarSiguiente()
    {
        if (_cola.Count == 0)
        { Console.WriteLine("✅ No hay pedidos pendientes."); return; }

        var pedido = _cola.Dequeue(); // extrae el más antiguo
        Console.WriteLine($"
🍕 COCINANDO: #{pedido.Id:D3} {pedido.Mesa} — {pedido.Cantidad}x {pedido.Pizza}");
        Console.WriteLine($"   Pedidos pendientes: {_cola.Count}");
    }

    // Ver el próximo pedido sin procesarlo
    public void VerProximo()
    {
        if (_cola.Count == 0) { Console.WriteLine("Cola vacía."); return; }
        var sig = _cola.Peek(); // ver sin extraer
        Console.WriteLine($"⏳ Próximo a cocinar: #{sig.Id:D3} {sig.Mesa} — {sig.Pizza}");
    }
}

🔵 Stack<T> — Pila (LIFO) BCL · System.Collections.Generic
📐 Definición técnica
Last In, First Out (LIFO). Push() agrega encima en O(1). Pop() extrae el tope en O(1). Peek() consulta sin extraer. El botón Ctrl+Z de cualquier programa usa un Stack internamente.
💡 Explicación sin jerga
Es una pila de platos sucios. El último plato que pusiste encima es el primero que lavas. Push() = poner un plato encima. Pop() = sacar el plato de arriba para lavarlo. Peek() = mirar cuál es el plato de arriba sin sacarlo todavía. En resumen: lo último que metiste es lo primero que sacas.

🎮 Tienda Gamer — Historial de compras del cliente. El último juego comprado aparece primero (como el historial de un navegador web).

¿Qué es LIFO?
Last In, First Out — Lo último que entró es lo primero que sale. Como una pila de platos: el que pusiste último encima es el primero que sacas. Push() = poner encima. Pop() = sacar el de arriba. Peek() = ver el de arriba sin sacarlo. El botón Ctrl+Z de cualquier programa usa un Stack internamente.
C# — Stack básico: historial de compras Tienda GamerBásico
// Stack<T> — El último que entró es el primero que sale (LIFO)
// Contexto: historial de compras del cliente en la tienda gamer
var historialCompras = new Stack<string>();

// Push(): agregar ENCIMA de la pila (igual que apilar platos)
historialCompras.Push("FIFA 25");       // comprado primero → queda abajo
historialCompras.Push("Elden Ring");    // comprado después → en el medio
historialCompras.Push("Spider-Man 2");  // comprado último → queda ARRIBA

Console.WriteLine($"Juegos comprados: {historialCompras.Count}"); // 3

// Peek(): ver el más reciente SIN sacarlo
Console.WriteLine($"Última compra: {historialCompras.Peek()}"); // Spider-Man 2

// Pop(): sacar el de arriba — el más reciente sale primero (LIFO)
Console.WriteLine("
Historial (más reciente primero):");
while (historialCompras.Count > 0)
{
    string juego = historialCompras.Pop(); // saca el de ARRIBA
    Console.WriteLine($"  🎮 {juego}");
}
// Salida (LIFO — inverso al orden de compra):
//   🎮 Spider-Man 2   ← comprado último, aparece PRIMERO
//   🎮 Elden Ring
//   🎮 FIFA 25        ← comprado primero, aparece ÚLTIMO
Console.ReadKey();

🛒 Supermercado — Historial de precios de un producto. El cajero puede deshacer el último cambio de precio (Ctrl+Z).

C# — Stack intermedio: deshacer cambios de precio SupermercadoIntermedio
// Stack para implementar Ctrl+Z (deshacer)
// Cada vez que cambia el precio, guardamos el anterior en la pila
// Al deshacer, sacamos el último precio guardado (el más reciente)

var historialPrecios = new Stack<(string Producto, decimal PrecioAnterior)>();
decimal precioActual = 1190m; // precio inicial del Arroz 1kg

Console.WriteLine($"Precio inicial: ${precioActual:N0}");

// Método que cambia el precio y guarda el anterior en el Stack
void CambiarPrecio(decimal nuevoPrecio)
{
    // Push: apilar el precio actual ANTES de cambiarlo
    historialPrecios.Push(("Arroz 1kg", precioActual));
    precioActual = nuevoPrecio;
    Console.WriteLine($"→ Precio cambiado a ${precioActual:N0}");
}

// Método que deshace el último cambio (Ctrl+Z)
void Deshacer()
{
    if (historialPrecios.Count == 0)
    { Console.WriteLine("⚠ Sin cambios para deshacer."); return; }

    // Pop: sacar el último precio guardado (el más reciente)
    var (prod, anterior) = historialPrecios.Pop();
    precioActual = anterior;
    Console.WriteLine($"↩ Deshecho — {prod} volvió a ${precioActual:N0}");
}

CambiarPrecio(1390m); // sube   → Stack tiene 1 elemento
CambiarPrecio(990m);  // baja   → Stack tiene 2 elementos
CambiarPrecio(1500m); // sube   → Stack tiene 3 elementos
Deshacer();           // vuelve a 990  ← saca el último (1500 era precio)
Deshacer();           // vuelve a 1390 ← saca el siguiente
Console.WriteLine($"Precio final: ${precioActual:N0}"); // 1390
Console.ReadKey();

// Sistema de auditoría y deshacer para la caja de la pizzería
// Registra cada acción del cajero y permite revertir la última
public class RegistroAcciones
{
    private readonly Stack<Accion> _historial = new();

    // Registrar una acción — la apila en el tope del Stack
    public void Registrar(string tipo, string detalle, decimal monto)
    {
        var accion = new Accion(DateTime.Now, tipo, detalle, monto);
        _historial.Push(accion);
        Console.WriteLine($"[{accion.Hora:HH:mm}] {tipo}: {detalle} ${monto:N0}");
    }

    // Deshacer la última acción (la del tope del Stack)
    public void DeshacerUltima()
    {
        if (_historial.Count == 0)
        { Console.WriteLine("⚠ Sin acciones para deshacer."); return; }
        var revertida = _historial.Pop(); // sacar la más reciente
        Console.WriteLine($"↩ DESHECHO: {revertida.Tipo} — {revertida.Detalle}");
    }

    // Ver el log completo sin destruir el Stack
    public void MostrarLog()
    {
        Console.WriteLine("
=== REGISTRO DEL TURNO ===");
        // foreach NO destruye el Stack — solo lo recorre visualmente
        foreach (var a in _historial)
            Console.WriteLine($"  [{a.Hora:HH:mm}] {a.Tipo,-15} {a.Detalle,-20} ${a.Monto:N0}");
        Console.WriteLine($"  Total acciones: {_historial.Count}");
    }
}

record Accion(DateTime Hora, string Tipo, string Detalle, decimal Monto);

// USAR el registro de acciones

📖 Glosario Técnico

Busca cualquier término. +2 XP por cada término explorado.

Interpolación de strings — $"..."C#

📐 Técnica: Mecanismo del compilador Roslyn para incrustar expresiones C# dentro de un literal de string usando llaves {}. Disponible desde C# 6.

⚠️ Regla: Siempre usar $"..." en vez de concatenación con +. Más legible y el compilador lo optimiza.

Ejemplo: $"Stock: {producto.Stock:N0} uds"

Array — Arreglo de tamaño fijoBCL

📐 Técnica: Estructura de datos lineal con posiciones contiguas en memoria. Tamaño fijo en creación. Acceso O(1) por índice. Clase base System.Array.

Uso: Cuando el número de elementos es conocido y fijo: 5 géneros de juegos, 7 días de la semana, 3 plataformas.

Java: Idéntico — int[] arr = new int[5] es igual en ambos.

List<T> — Colección dinámica tipadaColecciones

📐 Técnica: Implementa IList<T>. Array interno que se redimensiona automáticamente. Acceso O(1), búsqueda O(n), inserción al final O(1) amortizado.

Operaciones clave: Add(), Remove(), Find(), FindAll(), Sort(), Count

Java: Equivale a ArrayList<T> pero tipado sin cast.

Dictionary<K,V> — Mapa de clave-valorColecciones

📐 Técnica: Tabla hash. Cada clave es única. Búsqueda, inserción y eliminación O(1) promedio.

Operaciones clave: ContainsKey(), TryGetValue(), Add(), Remove()

Java: Equivale a HashMap<K,V>.

Queue<T> — Cola FIFOColecciones

📐 Técnica: First In, First Out. Enqueue() agrega al final, Dequeue() extrae del frente, Peek() consulta sin extraer.

Uso en Almacenes: Cola de atención a clientes, pedidos pendientes de procesar.

Stack<T> — Pila LIFOColecciones

📐 Técnica: Last In, First Out. Push() agrega encima, Pop() extrae el último, Peek() consulta sin extraer.

Uso en Almacenes: Historial de transacciones, funcionalidad deshacer en la app.

Expresión Lambda — p => condiciónC#

📐 Técnica: Función anónima inline. La sintaxis p => p.Stock > 0 significa "para cada p, evalúa p.Stock > 0". Se pasa como argumento a métodos como Find(), FindAll(), Sort().

Java: Equivale a las lambdas de Java 8+: p -> p.getStock() > 0.

ArrayList (Java) vs List<T> (C#)Previo

Diferencia clave: ArrayList de Java no es tipado — necesitas cast al extraer. List<T> de C# sí es tipado — el compilador garantiza el tipo.

Equivalencia: ArrayList<Producto> (Java) = List<Producto> (C#). Sin cast, sin riesgo de ClassCastException.

foreach vs for — ¿cuándo usar cada uno?BCL

foreach: Cuando solo necesitas el valor de cada elemento — más limpio y seguro.

for: Cuando necesitas el índice numérico (posición) o modificar el arreglo durante la iteración.

Regla: usa foreach por defecto, cambia a for solo si necesitas el índice.

TryGetValue vs ContainsKey — acceso seguroC#

❌ Peligroso: dict["clave"] lanza KeyNotFoundException si la clave no existe.

✅ Seguro: TryGetValue("clave", out var val) retorna false sin excepción.

Regla: siempre usar TryGetValue() en producción cuando no estás seguro de que la clave exista.
