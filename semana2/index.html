🔤 Interpolación de Strings

Antes de entrar a las colecciones, dominamos esto — lo usaremos en todos los ejemplos de la clase.

📐 Definición técnica
La interpolación de strings ($"...") es un mecanismo del compilador C# que permite incrustar expresiones dentro de un literal de string. El compilador de Roslyn la transforma en una llamada a string.Format() o StringBuilder optimizada. Disponible desde C# 6.0.
💡 En palabras simples
En vez de unir pedazos de texto con el operador + (concatenar), escribes el string completo y metes las variables directamente dentro de {llaves}. El código queda mucho más legible — y el compilador lo optimiza automáticamente.

❌ Concatenación vs ✅ Interpolación

❌ CONCATENACIÓN (evitar)
string msg = "Producto: " + nombre
+ " | Precio: $" + precio
+ " | Stock: " + stock
+ " unidades";
✅ INTERPOLACIÓN (usar siempre)
string msg = $"Producto: {nombre}
| Precio: ${precio:N0}
| Stock: {stock} unidades";
C# — Interpolación completa con formatos 
// La $ antes de la comilla activa la interpolación
// Dentro de {llaves} puedes poner cualquier expresión C#

string  nombre  = "Polera básica";
decimal precio  = 9990.50m;
int     stock   = 120;
bool    activo  = true;
DateTime fecha  = DateTime.Now;

// Interpolación básica: solo meter la variable
Console.WriteLine($"Nombre: {nombre}");

// Con formato numérico:  :N0 = número con separador de miles, sin decimales
Console.WriteLine($"Precio: ${precio:N0}");          // Precio: $9.991
Console.WriteLine($"Precio: ${precio:F2}");          // Precio: $9990,50

// Con alineación: {valor,-20} = alinear izquierda en 20 chars
Console.WriteLine($"{nombre,-25} ${precio,10:N0}"); // columnas bien alineadas

// Con expresión directa dentro de las llaves
Console.WriteLine($"¿Hay stock? {(stock > 0 ? "Sí" : "No")}");

// Con formato de fecha
Console.WriteLine($"Fecha: {fecha:dd/MM/yyyy HH:mm}"); // 19/03/2025 14:30

// Con D3: muestra el número con mínimo 3 dígitos (rellena con ceros)
Console.WriteLine($"ID: {42:D3}");  // ID: 042

// Interpolación multilínea (raw string — C# 11+)
string ficha = $"""
    ╔══════════════════════════╗
    ║  {nombre,-24}║
    ║  Precio: ${precio,14:N0}  ║
    ║  Stock:  {stock,14} uds  ║
    ╚══════════════════════════╝
    """;
Console.WriteLine(ficha);

📦 Arrays (Arreglos)

Estructura de datos de tamaño fijo. La base de todas las colecciones. Namespace: System (BCL base — siempre disponible, sin using extra).

📦 Array — Colección de tamaño fijo BCL · System
📐 Definición técnica
Estructura de datos lineal que almacena elementos del mismo tipo en posiciones contiguas de memoria. El tamaño se fija en la creación y no puede cambiar. Acceso en tiempo O(1) mediante índice. Los índices comienzan en 0. Clase base System.Array de la BCL.
💡 Analogía profesional
El array es como una estantería con un número fijo de casilleros. Cada casillero tiene un número (índice) y puedes ir directamente al casillero 47 sin revisar los anteriores. Perfecto cuando sabes exactamente cuántos elementos tendrás.
🔧 ¿Cuándo usar un Array?
✅ Tamaño conocido y fijo (ej: 12 meses, 7 días, 5 sucursales) · ✅ Acceso frecuente por índice · ✅ Datos homogéneos · ❌ NO usar cuando la cantidad de elementos varía en tiempo de ejecución.

Declarar, inicializar y recorrer un array simple.

C# — Array básico de preciosBásico
// DECLARAR un array: tipo[] nombre = new tipo[tamaño];
// El tamaño se define UNA VEZ y no cambia
decimal[] precios = new decimal[5]; // array de 5 decimales (todos en 0)

// ASIGNAR valores por índice (siempre empieza en 0)
precios[0] = 9990m;   // primer elemento  → índice 0
precios[1] = 24990m;  // segundo elemento → índice 1
precios[2] = 4990m;   // tercer elemento  → índice 2
precios[3] = 49990m;  // cuarto elemento  → índice 3
precios[4] = 12990m;  // quinto elemento  → índice 4 (último)

// INICIALIZAR directamente con valores (forma corta)
string[] tallas = { "XS", "S", "M", "L", "XL", "XXL" };

// LEER un elemento específico
Console.WriteLine($"Primer precio: ${precios[0]:N0}");  // $9.990
Console.WriteLine($"Última talla:  {tallas[5]}");       // XXL

// RECORRER con foreach — la forma más limpia
// foreach no necesita índice, recorre todos los elementos
Console.WriteLine("\nLista de precios:");
foreach (decimal precio in precios)
{
    Console.WriteLine($"  ${precio:N0}");
}

// RECORRER con for — cuando necesitas el índice
Console.WriteLine("\nTallas disponibles:");
for (int i = 0; i < tallas.Length; i++) // .Length = número de elementos
{
    Console.WriteLine($"  [{i}] {tallas[i]}");
}

Arrays multidimensionales y operaciones útiles de System.Array.

C# — Array 2D: stock por sucursal y productoIntermedio
// ARRAY BIDIMENSIONAL: [filas, columnas]
// Útil para tablas de datos (como una hoja de cálculo)
// Aquí: 3 sucursales × 4 productos = tabla de stock
int[,] stock = new int[3, 4]  // 3 sucursales, 4 productos
{
    { 120, 45, 80, 200 },   // sucursal 0 (Santiago Centro)
    {  60, 30, 90,  50 },   // sucursal 1 (Providencia)
    {  40, 70, 20, 110 },   // sucursal 2 (Ñuñoa)
};

string[] sucursales = { "Santiago C.", "Providencia", "Ñuñoa" };
string[] productos  = { "Polera", "Jeans", "Chaqueta", "Calcetines" };

// RECORRER array 2D con dos for anidados
Console.WriteLine("=== STOCK POR SUCURSAL ===");
Console.Write($"{"",14}"); // espacio para los nombres de columna
foreach (string p in productos)
    Console.Write($"{p,12}");  // encabezado de columnas alineado
Console.WriteLine();

for (int s = 0; s < stock.GetLength(0); s++)  // .GetLength(0) = filas
{
    Console.Write($"{sucursales[s],-14}");
    for (int p = 0; p < stock.GetLength(1); p++) // .GetLength(1) = columnas
    {
        Console.Write($"{stock[s,p],12}");  // alinear a derecha en 12 chars
    }
    Console.WriteLine();
}

// OPERACIONES ÚTILES de System.Array
int[] stocks1D = { 120, 45, 80, 200, 60 };

Array.Sort(stocks1D);      // ordena de menor a mayor
Array.Reverse(stocks1D);   // invierte el orden (ahora de mayor a menor)

Console.WriteLine($"\nMáximo stock:  {stocks1D[0]}");   // primer tras reverse
Console.WriteLine($"Mínimo stock:  {stocks1D[^1]}");   // ^1 = último elemento
Console.WriteLine($"Total filas:   {stock.GetLength(0)}");
Console.WriteLine($"Total columnas:{stock.GetLength(1)}");

C# — Informe de inventario con arraysProfesional
// Escenario real: Almacenes Por Ahí necesita un informe
// que muestre el stock de cada sucursal y detecte
// las sucursales con stock crítico (< umbral)

string[] sucursales = { "Santiago C.", "Providencia", "Ñuñoa", "La Florida" };
decimal[] precios   = { 9990m, 24990m, 4990m, 49990m };
string[]  productos = { "Polera", "Jeans", "Calcetines", "Chaqueta" };

// Tabla de stock: sucursales (filas) × productos (columnas)
int[,] stock = {
    { 120, 45,  8,  30 },  // Santiago C.
    {  60, 30, 200,   5 },  // Providencia ← Chaqueta crítica
    {  40,  7,  90,  15 },  // Ñuñoa ← Jeans crítico
    { 200, 80, 150,  25 },  // La Florida
};

const int UMBRAL_CRITICO = 10;  // menos de 10 unidades = alerta

Console.WriteLine("╔══════════════════════════════════════════════════════╗");
Console.WriteLine("║        ALMACENES POR AHÍ — INFORME DE STOCK          ║");
Console.WriteLine("╚══════════════════════════════════════════════════════╝");

// Encabezado de columnas usando interpolación
Console.Write($"\n{"SUCURSAL",-14}");
for (int p = 0; p < productos.Length; p++)
    Console.Write($"{productos[p],12}");
Console.Write($"{"TOTAL $",14}\n{"",14}");
Console.WriteLine(new string('─', 66));

decimal totalGlobal = 0;

for (int s = 0; s < sucursales.Length; s++)
{
    decimal totalSucursal = 0;
    Console.Write($"{sucursales[s],-14}");

    for (int p = 0; p < productos.Length; p++)
    {
        int unidades = stock[s, p];
        // Marcar visualmente si el stock está bajo el umbral
        string indicador = unidades < UMBRAL_CRITICO ? "⚠" : " ";
        Console.Write($"{indicador}{unidades,10}");

        totalSucursal += precios[p] * unidades; // valor = precio × unidades
    }

    Console.WriteLine($"  ${totalSucursal,10:N0}");
    totalGlobal += totalSucursal;
}

Console.WriteLine(new string('─', 66));
Console.WriteLine($"{"VALOR TOTAL INVENTARIO",-50} ${totalGlobal:N0}");
Console.WriteLine("\n⚠ = Stock crítico (menos de 10 unidades)");

📋 List<T> — La Colección Estrella

Colección dinámica de la BCL. La más usada en C# del día a día. Namespace: System.Collections.Generic.

📋 List<T> — Colección dinámica tipada BCL · System.Collections.Generic
📐 Definición técnica
List<T> implementa IList<T> e ICollection<T>. Internamente usa un array que se redimensiona automáticamente (duplica capacidad al superar el límite). Acceso aleatorio O(1), inserción al final O(1) amortizado, búsqueda lineal O(n).
💡 En palabras simples
Como un array pero que crece o se encoge solo. No necesitas saber de antemano cuántos elementos tendrás. Es el ArrayList tipado de Java: no admite tipos incorrectos y no necesitas hacer cast.
🔧 Import necesario
using System.Collections.Generic; — En .NET 8 con ImplicitUsings=enable en el .csproj, ya está incluido automáticamente. Si no: agrégalo al inicio del archivo.
C# — List<T> básicoBásico
using System.Collections.Generic; // importar el namespace de la BCL

// CREAR una lista vacía — el T es el tipo de los elementos
List<string> categorias = new List<string>();

// ADD: agregar elementos al final — tiempo O(1)
categorias.Add("Poleras");
categorias.Add("Jeans");
categorias.Add("Chaquetas");
categorias.Add("Calzado");

// COUNT: número de elementos (equivale a .size() de Java)
Console.WriteLine($"Categorías registradas: {categorias.Count}");  // 4

// ACCEDER por índice (igual que array)
Console.WriteLine($"Primera categoría: {categorias[0]}"); // Poleras

// RECORRER con foreach
Console.WriteLine("\nCatálogo de categorías:");
foreach (string cat in categorias)
    Console.WriteLine($"  • {cat}");

// REMOVE: eliminar un elemento por valor
categorias.Remove("Calzado");

// INSERT: insertar en posición específica
categorias.Insert(1, "Accesorios"); // insertar en índice 1

// CONTAINS: verificar si existe el elemento
bool tieneJeans = categorias.Contains("Jeans");
Console.WriteLine($"¿Hay Jeans? {tieneJeans}");  // True

// CLEAR: vaciar la lista completamente
categorias.Clear();
Console.WriteLine($"Elementos tras Clear: {categorias.Count}");  // 0
C# — List<T> con objetos y búsquedasIntermedio
// Lista de objetos Producto — la BCL acepta cualquier tipo
var inventario = new List<Producto>();

// Agregar objetos completos
inventario.Add(new Producto(1, "Polera básica",   9990m,  120));
inventario.Add(new Producto(2, "Jeans skinny",     24990m, 45));
inventario.Add(new Producto(3, "Chaqueta invierno", 49990m, 0));
inventario.Add(new Producto(4, "Calcetines x3",    4990m,  200));

// FIND: devuelve el PRIMERO que cumple la condición (lambda)
// p => es la "función anónima": por cada p en la lista, evalúa la condición
Producto? encontrado = inventario.Find(p => p.Id == 2);
if (encontrado != null)
    Console.WriteLine($"Encontrado: {encontrado.Nombre}");

// FINDALL: devuelve TODOS los que cumplen la condición
List<Producto> conStock = inventario.FindAll(p => p.Stock > 0);
Console.WriteLine($"Productos con stock: {conStock.Count}");  // 3

// SORT: ordenar con un comparador lambda
// a y b son dos elementos — CompareTo compara los valores
inventario.Sort((a, b) => a.Precio.CompareTo(b.Precio)); // menor a mayor
Console.WriteLine("\n=== Ordenado por precio ===");
foreach (var p in inventario)
    Console.WriteLine($"  [{p.Id:D3}] {p.Nombre,-22} ${p.Precio:N0}");

// REMOVEALL: eliminar todos los que cumplen la condición
int eliminados = inventario.RemoveAll(p => p.Stock == 0); // sin stock
Console.WriteLine($"\nProductos eliminados (sin stock): {eliminados}");

record Producto(int Id, string Nombre, decimal Precio, int Stock);
C# — Gestión completa de inventario con List<T>Profesional
// Clase de servicio que encapsula todas las operaciones
// sobre la lista de productos — patrón común en proyectos .NET
public class InventarioService
{
    private readonly List<Producto> _productos = new();
    private int _proximoId = 1;

    // Agregar validando que no exista nombre duplicado
    public bool Agregar(string nombre, decimal precio, int stock)
    {
        // Any: retorna true si al menos UNO cumple la condición
        bool existe = _productos.Any(p => p.Nombre.Equals(nombre,
            StringComparison.OrdinalIgnoreCase));
        if (existe) return false; // duplicado → rechazar

        _productos.Add(new Producto(_proximoId++, nombre, precio, stock));
        return true;
    }

    // Buscar por texto parcial en el nombre (búsqueda tipo "contiene")
    public List<Producto> Buscar(string texto)
        => _productos.FindAll(p =>
            p.Nombre.Contains(texto, StringComparison.OrdinalIgnoreCase));

    // Informe resumen con interpolación bien formateada
    public void Informe()
    {
        Console.WriteLine($"\n{"ID",-5} {"NOMBRE",-25} {"PRECIO",10} {"STOCK",7} {"ESTADO",10}");
        Console.WriteLine(new string('─', 58));

        foreach (var p in _productos)
        {
            string estado = p.Stock == 0 ? "⛔ AGOTADO"
                           : p.Stock < 10 ? "⚠ CRÍTICO"
                           : "✅ OK";
            Console.WriteLine(
                $"{p.Id:D3,-5} {p.Nombre,-25} ${p.Precio,9:N0} {p.Stock,7} {estado}");
        }

        // Calcular estadísticas recorriendo la lista
        decimal valorTotal = 0;
        int sinStock = 0;
        foreach (var p in _productos)
        {
            valorTotal += p.Precio * p.Stock;
            if (p.Stock == 0) sinStock++;
        }

        Console.WriteLine(new string('─', 58));
        Console.WriteLine($"Total productos: {_productos.Count} | " +
                          $"Sin stock: {sinStock} | Valor inv: ${valorTotal:N0}");
    }
}

// USO del servicio
var svc = new InventarioService();
svc.Agregar("Polera básica", 9990m, 120);
svc.Agregar("Jeans skinny",  24990m, 45);
svc.Agregar("Chaqueta",      49990m, 0);
svc.Informe();

🗂️ Dictionary<K,V> — Búsqueda por Clave

Colección de pares clave-valor. Acceso en tiempo O(1) por clave. Namespace: System.Collections.Generic.

🗂️ Dictionary<K,V> — Mapa de clave-valor BCL · System.Collections.Generic
📐 Definición técnica
Implementa IDictionary<K,V> usando una tabla hash internamente. Cada clave es única. Búsqueda, inserción y eliminación en tiempo O(1) promedio. El tipo K debe implementar GetHashCode() y Equals(). Equivale a HashMap de Java.
💡 En palabras simples
Como un diccionario real: buscas una palabra (clave) y obtienes su definición (valor) directo — sin leer todo el libro. Perfecto para: buscar un producto por ID, buscar un cliente por RUT, o el precio de un artículo por su código.
C# — Dictionary básicoBásico
// CREAR: Dictionary<clave, valor>
// string = tipo de la clave | decimal = tipo del valor
Dictionary<string, decimal> precios = new()
{
    { "Polera",    9990m  },   // clave → valor
    { "Jeans",     24990m },
    { "Chaqueta",  49990m },
    { "Calcetines",4990m  },
};

// ACCEDER por clave — directo, O(1)
Console.WriteLine($"Precio polera: ${precios["Polera"]:N0}");

// AGREGAR un nuevo par
precios["Gorra"] = 7990m;  // si no existe, lo crea; si existe, lo actualiza

// VERIFICAR si existe la clave ANTES de acceder (evita excepción)
string buscar = "Zapatillas";
if (precios.ContainsKey(buscar))
    Console.WriteLine($"{buscar}: ${precios[buscar]:N0}");
else
    Console.WriteLine($"'{buscar}' no está en el catálogo");

// TryGetValue: forma segura — no lanza excepción si no existe
if (precios.TryGetValue("Jeans", out decimal precioJeans))
    Console.WriteLine($"Jeans cuesta: ${precioJeans:N0}");

// RECORRER todos los pares clave-valor
Console.WriteLine("\nCatálogo de precios:");
foreach (KeyValuePair<string, decimal> par in precios)
    Console.WriteLine($"  {par.Key,-15} ${par.Value:N0}");

// ELIMINAR por clave
precios.Remove("Gorra");
Console.WriteLine($"\nTotal artículos: {precios.Count}");
C# — Dictionary con objetos como valorIntermedio
// Clave: string (RUT del cliente) → Valor: objeto Cliente completo
// Permite encontrar un cliente en O(1) por su RUT
var clientesPorRUT = new Dictionary<string, Cliente>();

// Agregar clientes
clientesPorRUT["12.345.678-9"] = new Cliente("Ana López", "ana@mail.com");
clientesPorRUT["98.765.432-1"] = new Cliente("Juan Pérez", "juan@mail.com");
clientesPorRUT["45.678.901-2"] = new Cliente("María Torres", "maria@mail.com");

// Buscar cliente por RUT — O(1) sin recorrer toda la lista
string rut = "12.345.678-9";
if (clientesPorRUT.TryGetValue(rut, out Cliente? cliente))
    Console.WriteLine($"Cliente encontrado: {cliente.Nombre} — {cliente.Email}");

// Recorrer usando deconstrucción (C# moderno)
Console.WriteLine("\n=== DIRECTORIO DE CLIENTES ===");
foreach (var (rutKey, cli) in clientesPorRUT)  // deconstrucción del par
    Console.WriteLine($"  {rutKey,-15} {cli.Nombre,-20} {cli.Email}");

// Dictionary como contador (frecuencia de ventas por categoría)
var ventasPorCategoria = new Dictionary<string, int>();
string[] ventasHoy = { "Polera", "Jeans", "Polera", "Chaqueta", "Polera" };

foreach (string cat in ventasHoy)
{
    // Si la clave existe, incrementa; si no, inicia en 1
    if (ventasPorCategoria.ContainsKey(cat))
        ventasPorCategoria[cat]++;
    else
        ventasPorCategoria[cat] = 1;
}

Console.WriteLine("\n=== VENTAS POR CATEGORÍA HOY ===");
foreach (var (cat, total) in ventasPorCategoria)
    Console.WriteLine($"  {cat,-15}: {total} unidades");

record Cliente(string Nombre, string Email);
C# — Sistema de caché y lookup con DictionaryProfesional
// Escenario real: sistema de punto de venta
// El cajero escanea el código del producto y necesita el precio
// al instante — el Dictionary funciona como lookup table O(1)

public class PuntoDeVenta
{
    // Clave: código SKU del producto | Valor: (nombre, precio)
    private readonly Dictionary<string, (string Nombre, decimal Precio)> _catalogo;

    // Historial de transacciones: hora → lista de ítems comprados
    private readonly Dictionary<DateTime, List<string>> _historial = new();

    public PuntoDeVenta()
    {
        // Inicializar catálogo con sintaxis de colección moderna (C# 9+)
        _catalogo = new()
        {
            ["POL-001"] = ("Polera básica",   9990m),
            ["JNS-042"] = ("Jeans skinny",     24990m),
            ["CHA-007"] = ("Chaqueta invierno", 49990m),
            ["CAL-099"] = ("Calcetines x3",    4990m),
        };
    }

    // Procesar una venta: recibe lista de SKUs escaneados
    public decimal ProcesarVenta(List<string> skusEscaneados)
    {
        decimal total = 0;
        var itemsVenta = new List<string>();

        Console.WriteLine("\n=== BOLETA ALMACENES POR AHÍ ===");

        foreach (string sku in skusEscaneados)
        {
            // TryGetValue: forma segura de buscar — no lanza excepción
            if (_catalogo.TryGetValue(sku, out var item))
            {
                total += item.Precio;
                itemsVenta.Add($"{sku}: {item.Nombre}");
                Console.WriteLine($"  {item.Nombre,-25} ${item.Precio:N0}");
            }
            else
                Console.WriteLine($"  SKU '{sku}' no encontrado ⚠️");
        }

        Console.WriteLine($"{"",30}──────────");
        Console.WriteLine($"  {"TOTAL",-25} ${total:N0}");

        // Guardar en historial con la hora actual como clave
        _historial[DateTime.Now] = itemsVenta;

        return total;
    }

    // Mostrar estadísticas del día
    public void ResumenDia()
    {
        Console.WriteLine($"\nTransacciones hoy: {_historial.Count}");
        Console.WriteLine($"Artículos en catálogo: {_catalogo.Count}");
    }
}

// USO
var pos = new PuntoDeVenta();
pos.ProcesarVenta(new List<string> { "POL-001", "JNS-042", "XYZ-999" });
pos.ResumenDia();

🔁 Queue<T> y Stack<T>

Colecciones especializadas para casos de uso específicos. Namespace: System.Collections.Generic.

🟡 Queue<T> — FIFO BCL
📐 Técnica
First In, First Out. Enqueue() agrega al final, Dequeue() extrae del frente. Ideal para colas de espera, procesamiento de pedidos, mensajes.
C# — Cola de pedidos
// Cola de pedidos — el primero que llega, primero que se atiende
var colaPedidos = new Queue<string>();

// Enqueue: agregar al FINAL de la cola
colaPedidos.Enqueue("Pedido #001 — Ana");
colaPedidos.Enqueue("Pedido #002 — Juan");
colaPedidos.Enqueue("Pedido #003 — María");

Console.WriteLine($"Pedidos en espera: {colaPedidos.Count}");

// Peek: ver el próximo sin extraerlo
Console.WriteLine($"Próximo: {colaPedidos.Peek()}");

// Dequeue: extraer el PRIMERO (FIFO)
while (colaPedidos.Count > 0)
{
    string pedido = colaPedidos.Dequeue();
    Console.WriteLine($"Procesando: {pedido}");
}
🔵 Stack<T> — LIFO BCL
📐 Técnica
Last In, First Out. Push() agrega encima, Pop() extrae desde arriba. Ideal para historial de acciones, deshacer cambios, navegación.
C# — Historial de acciones
// Stack: historial de acciones — Ctrl+Z deshacer
var historial = new Stack<string>();

// Push: agregar encima de la pila
historial.Push("Agregó Polera × 2");
historial.Push("Aplicó descuento 15%");
historial.Push("Cambió método de pago");

// Peek: ver la última acción sin quitarla
Console.WriteLine($"Última acción: {historial.Peek()}");

// Pop: deshacer la última acción (LIFO)
Console.WriteLine("\nDeshaciendo:");
while (historial.Count > 0)
{
    string accion = historial.Pop();
    Console.WriteLine($"↩ Deshecho: {accion}");
}

⚖️ ¿Qué colección usar?

La decisión correcta depende del caso de uso. Esta tabla es tu referencia rápida.

Colección Cuándo usarla Cuándo NO usarla En Almacenes Por Ahí
Array T[] Tamaño fijo y conocido. Acceso por índice constante. Cantidad variable de elementos. Las 12 sucursales, los 7 días de la semana.
List<T> Lista dinámica de cualquier tamaño. El 90% de los casos. Búsqueda frecuente por clave (usa Dictionary). Lista de productos, clientes, ventas del día.
Dictionary<K,V> Buscar por clave única en O(1). Claves no duplicadas. Datos sin clave natural o con duplicados. Buscar producto por SKU, cliente por RUT.
Queue<T> Procesamiento en orden de llegada (FIFO). Cuando necesitas acceso aleatorio. Cola de atención a clientes, pedidos pendientes.
Stack<T> Historial, deshacer acciones, navegación (LIFO). Procesamiento ordenado por llegada. Historial de transacciones, deshacer en UI.

📖 Glosario Técnico

Busca cualquier término. +2 XP por cada término explorado.

Interpolación de strings — $"..."C#

📐 Técnica: Mecanismo del compilador Roslyn para incrustar expresiones C# dentro de un literal de string usando llaves {}. Disponible desde C# 6.

⚠️ Regla: Siempre usar $"..." en vez de concatenación con +. Más legible y el compilador lo optimiza.

Ejemplo: $"Stock: {producto.Stock:N0} uds"

Array — Arreglo de tamaño fijoBCL

📐 Técnica: Estructura de datos lineal con posiciones contiguas en memoria. Tamaño fijo en creación. Acceso O(1) por índice. Clase base System.Array.

Uso: Cuando el número de elementos es conocido y fijo (12 sucursales, 7 días).

Java: Idéntico — int[] arr = new int[5] es igual en ambos.

List<T> — Colección dinámica tipadaColecciones

📐 Técnica: Implementa IList<T>. Array interno que se redimensiona automáticamente. Acceso O(1), búsqueda O(n), inserción al final O(1) amortizado.

Operaciones clave: Add(), Remove(), Find(), FindAll(), Sort(), Count

Java: Equivale a ArrayList<T> pero tipado sin cast.

Dictionary<K,V> — Mapa de clave-valorColecciones

📐 Técnica: Tabla hash. Cada clave es única. Búsqueda, inserción y eliminación O(1) promedio.

Operaciones clave: ContainsKey(), TryGetValue(), Add(), Remove()

Java: Equivale a HashMap<K,V>.

Queue<T> — Cola FIFOColecciones

📐 Técnica: First In, First Out. Enqueue() agrega al final, Dequeue() extrae del frente, Peek() consulta sin extraer.

Uso en Almacenes: Cola de atención a clientes, pedidos pendientes de procesar.

Stack<T> — Pila LIFOColecciones

📐 Técnica: Last In, First Out. Push() agrega encima, Pop() extrae el último, Peek() consulta sin extraer.

Uso en Almacenes: Historial de transacciones, funcionalidad deshacer en la app.

Expresión Lambda — p => condiciónC#

📐 Técnica: Función anónima inline. La sintaxis p => p.Stock > 0 significa "para cada p, evalúa p.Stock > 0". Se pasa como argumento a métodos como Find(), FindAll(), Sort().

Java: Equivale a las lambdas de Java 8+: p -> p.getStock() > 0.

ArrayList (Java) vs List<T> (C#)Previo

Diferencia clave: ArrayList de Java no es tipado — necesitas cast al extraer. List<T> de C# sí es tipado — el compilador garantiza el tipo.

Equivalencia: ArrayList<Producto> (Java) = List<Producto> (C#). Sin cast, sin riesgo de ClassCastException.

foreach vs for — ¿cuándo usar cada uno?BCL

foreach: Cuando solo necesitas el valor de cada elemento — más limpio y seguro.

for: Cuando necesitas el índice numérico (posición) o modificar el arreglo durante la iteración.

Regla: usa foreach por defecto, cambia a for solo si necesitas el índice.

TryGetValue vs ContainsKey — acceso seguroC#

❌ Peligroso: dict["clave"] lanza KeyNotFoundException si la clave no existe.

✅ Seguro: TryGetValue("clave", out var val) retorna false sin excepción.

Regla: siempre usar TryGetValue() en producción cuando no estás seguro de que la clave exista.
